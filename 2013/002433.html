The software project unbound provides a recursive DNS resolver.
I have been testing it for a few days.
I designed the spell to install it so that unbound would run from the
rootfs.
In contrast bind runs from the root file system.
The reason for the difference is so that a single unbound process
can provide resolution for any number of root file systems.
40,499,146 bind
1,855,826 unbound

The obvious benefit of unbound is that it is more than a magnitude smaller
than bind.
However, the configuration the slightly modified default configuration for
it,
might be slightly less than ideal compared to bind.
Anyone who wants to help is invited to test unbound and make recommendations
for default configuration file changes that could help improve performance
and minimize memory usage.

Other recommendations for minimalistic recursive DNS resolvers are invited.
