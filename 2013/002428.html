Boxes that updated are probably missing
/etc/init.d/{rootname,veth-setup}
These important files should be provided by init-scripts.
Unfortunately, somehow they were omitted from being added to the svn.
Thankfully, Evert spotted and reported it.

I have made corrections and rolled new grimoire tarballs
that replace the previous 4 days worth of grimoire tarballs.
So the solution should be as simple as:

# rm -fr /aux/can/source/grimoire; au begin grimoire; au scry
grimoire; au begin init-scripts

However, if a box already boots without those init-scripts
the the network adapter will not be available in the root file system.
Perhaps the easiest fix for that box might be something like
log into the root file system.
edit /etc/init.d/namespace
locate the -imnpu and change it to -impu
Therefore, on the next boot the network namespace will not be unshared.
Then issue shutdown -r now.
When the box reboots networking should work from the root file system.

# rm -fr /aux/can/source/grimoire; au begin grimoire; au scry
grimoire; au begin init-scripts

And when init-scripts casts it will install the missing init-scripts
and also uncustomized the modification made to /etc/init.d/namespace.

Init-scripts are provided in /etc/init.d/ which is within the /etc/ hierarchy.
However, init-scripts are not configuration files.
Customized configuration files are preserved across updates.
Customization of init-scripts are overwritten.

Hopefully, those instructions will assist
everyone with circumventing the only rough aspect of transition.
Sorry for the inconvenience.
