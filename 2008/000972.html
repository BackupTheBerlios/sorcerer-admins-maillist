There is a major drawback using pbzip2 as decompression program if 
sourcefile is compressed with a bzip2 in beginning.
It's not fast at all. Decompression is only fast if it's compressed with 
pbzip2, it organizes compressed file so it can be decompressed fast way, 
but if it was compressed using bzip2 there is no speedup to decompress 
with pbzip2.

Same test environment as in last message, but this time source file was 
compressed using bzip2:

pbzip2 decompress, 2 core, bzip2 created source
real    0m20.371s
user    0m19.005s
sys     0m0.672s

pbzip2 decompress, 4 core, bzip2 created source
real    0m19.598s
user    0m18.865s
sys     0m0.640s

And what if source file is compressed with pbzip2:
pbzip2 decompress, 2 core, pbzip2 created source
real    0m5.991s
user    0m11.277s
sys     0m0.272s

pbzip2 decompress, 4 core, pbzip2 created souce
real    0m3.961s
user    0m13.849s
sys     0m0.376s
