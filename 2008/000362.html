About KDE 4.0 and sound.
It appears to use a wrapper called phonon
which is included in KDE source tarballs.
However, phonon is not a sound server.
It is a wrapper that fits between the application and the sound server.
That way each application has a standard interface provided by phonon
while phonon handles the translation to a real sound server.
Phonon is supposed to be able to support several sound servers
to include the built in ALSA software mixer called dmix.
However, I have yet to discover a way to make it use dmix.
It can use jackit, enlightenment sound server, NAS
and a few other sound servers that are either broken,
old and obsolete and unmaintained.
Phonon is also supposed to be able to use xine-lib
as something like a sound server/mixer.
However, I have not yet discovered how to do this.
Perhaps it requires another software component that wraps xine-lib?
