Sorry about that.
p7zip always creates files mode 0600.
Every other POSIX compliant program respects umask.
p7zip does not.
I forget to fix it.
The files are now readable.

I did not create a sdelta3 port for windoze.
Seems like the demand for sources
for POSIX compliant software on
windoze boxes is very low.
It should be possible to create the port.
Adjust a few includes and
the location of the temporary file
and it might just work.

The BSD port was troublesome
because BSD's malloc is half broken.
But when sdelta switched to using mmap
the BSD port started working almost as good as Linux.
And sdelta also compiles on Solaris.

Of course Linux, BSD, and Solaris are more alike than windoze.
sdelta3 cygwin would probably be an easier port than
creating a native win32 port.
If you have a win32 compiler and an itch to scratch
you are welcome to try making it compile.
sdelta3 is actually a very simple program although
it swallows files whole with mmap which makes it
in that respect unusual.
