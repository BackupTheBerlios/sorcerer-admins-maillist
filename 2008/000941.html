I agree and disagree.
Having Python 3.0 concurrently installed at the moment
may cause a problem if Python modules are installed
for only version 3.0 and not the older Python while
other programs require the older.
Somehow I expect that the Python 3.0 update will be a mess when it happens.
I am not even content to hide it on /opt/Python
unless something absolutely requires Python 3.0
For those reasons I added it as a latest version progress
rather than creating another spell for it that even if it did work
would be a pain to clean up later.

SAs have expressed a desire that Sorcerer
behave more ?stable? even at the cost of not using current
stable releases of software.
If I do not use the "latest" version progress for this and also
for libraries where appropriate
then the effort goes to waste.
