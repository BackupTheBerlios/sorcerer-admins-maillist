Unfortunately...
while $LINES does exist from an interactive shell
it is not set when running a script.

$ cat /tmp/runme
#!/bin/bash

echo LINES=$LINES

Wed May 28 16:08:43
$ /tmp/runme
LINES=

So much for a variable length waiting list?

How many waiting spells do you want it to display?
Pre-empting cast's invocation of spells using
# augur go is not always a wise decision.
If a solo spell has cast in a only one vice grip
then it is usually a good idea for it to be compiling
only that one spell at that moment.

sorcery-md5sum sorcery-defrag binutils gcc sorcery-sentinel
coreutils and other solo spells are safest to compile solo.

Also augur scry does not display spells
that are waiting to download.
It merely displays a count for those spells.
Therefore, spells shown on augur scry are either
casting waiting or downloading.

Downloading can happen so quickly that augur scry
may not have a chance to display the percent of the
source which is acquired.
Generally, when downloading augur scry merely provides
an indication of what sources may have stalled while downloading.
If curl has proper limiter parameters in it's configuration file
then curl itself will automatically abort stalled downloads
which is better than leech hanging.
