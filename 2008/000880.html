All things considered, so long as such a rollback were announced ahead
of time (and not considered mandatory), it should be fairly trivial
for a given SA to dump out his or her tables (my preferred settings
are mysqldump -q -C -c databasename)  beforehand in the event that
there is a storage format problem.  I pretty much do this all day long
for various reasons (yay R&D), and I've seen only very minor hiccups.
It would require due diligence on the part of the SA in question, of
course, but so would sticking with 5.1 for the time being.

That said, Monty does state that so long as the 5.1-specific features
are not in use, then one shouldn't have any more problems than one
would with 5.0.

Another quick aside:  this is a great time to upgrade to postgresql :-)
