Anyone using using tcp_wrappers to firewall sshd
instead of iptables please send me an email.

If tcp_wrappers has an appropriate use
then that use would be for controlling access
to services assigned random ports by portmapper.

However, for services that run on dedicated ports
iptables is a better firewall choice.
Why?
iptables have more methods for controlling access than tcp_wrappers.
Executing firewalling in kernel space is faster than userspace.
And...

-r--r--r--  1 wietse  wheel  138700 May  4  2004 tcp_wrappers_7.6-ipv6.4.tar.gz

4 years old!

The only grimoire supported programs
with absolute requirements on tcp_wrappers are:
DenyHosts and portmap

However, the following programs have an optional requirement on tcp_wrappers:
apcupsd exim gdm inetutils openssh prelude-manager stunnel vsftpd

Fortunately, all the programs with optional requirements on tcp_wrappers
listen on dedicated ports and can be firewalled using iptables.

Currently nfs-utils requires portmap
only because init-scripts for NFS services do not specify port numbers,
but instead allow portmapper to provide port numbers.
However, all of the nfs required services support the parameter -p
to enable the service to bind to a specified port number
instead of using the random port assigned by the portmapper.
Therefore, NFS can be configured to work without portmapper
and listen on dedicated ports and be firewalled by iptables.

Does anyone have an objection to static configuration of ports involving NFS?
Does anyone have it already configured with static port numbers?
What port numbers should it be assigned?
