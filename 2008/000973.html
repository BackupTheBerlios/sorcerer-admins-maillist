Thank you for the results.
However, it is important to consider the file size.
How large of a file is being compressed?
How many 900K blocks please?
For such a large compression time I am expecting
the file must be close to the linux or firefox source tarball sizes?

I am surprised if pbzip2 must be used as the compression program
in order to gain a commensurate speedup for decompression.
However, if results show it to be so then it is so.
In that case staying with bzip2 may be better.
Chances are we will encounter few if any source tarballs
that were compressed with pbzip2.
