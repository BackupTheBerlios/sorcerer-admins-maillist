hi all,
I use git for some time now to have my how project tracked. recently 
after an upgrade I had some of my /etc files replaced as expected (I set 
sorcery to replace my /etc/ file and backup the old)
I done the same as usual : a diff between the new and backup and since 
only the runlevel changed (as usual) I replaced the new file by the 
backup... right after I thought that a simple merge would have 
sufficed... and immediatly thought of git
imagine you have /etc managed by git
2 main branches : official and custom
upon install  a 
git checkout official
the new files are copied
git commit -a -m "cast of yy/mm/dd" #maybe some stuff to do here to have 
the new files added to the repository)
git checkout custom #switch back to the custom modified files
git merge official #brings back configuration modification into our file

this will then merge the change if no conflicts.....
this also can avoid maintaining a /etc/sorcery/log/etc tree (that grows 
and grows all time on my box, with kde files etc)
what do you think about it?
