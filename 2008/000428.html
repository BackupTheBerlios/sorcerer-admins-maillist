Illegal instructions in p7zip ccache which

So far no insurmountable problems have been found.
However, finding those non i586 compatible executables
has impelled me to begin a rebuild of the i586 IA32 I/R
on a pentium mmx 233 Mhz.
Might be a few days before I can roll it out.

If you install from the current Sorcerer IA32 I/R be certain to
# dispel --unsafe ccache; augur forget ccache; rm -f
/var/cache/archive/ccache*; augur index; cast which p7zip

That was all I had to do thus far to hop around the problematic executables.
However, I have no simple way to tell which are good and which are not.
Therefore, I am recompiling everything installed before the previous
cast of gcc.
gcc compiles without ccache, bootstraps itself, and also removes the previous
ccache cache of compiled object files.
Therefore, if the continuation of the problem was due to the presence and re-use
of ccache cached object files then this should be sufficient to
eliminate the problem.

Sorry, for the inconvenience.
I will post when the next bug fixed IA32 I/R is available for download.
And you can still use the current IA32 I/R if you circumvent the problems
as I did above or are nstalling it on an Athlon-XP
