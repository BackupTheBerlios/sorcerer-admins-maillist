It has:

current(){ echo NVIDIA-Linux-x86 | pipe_queue $CAST_QUEUE com; }

Therefore, on an
# augur newer
The correct spell NVIDIA-Linux-x86 would be injected into the cast queue.

The NVIDIA-Linux-x86 spell als provides
/etc/init.d/{agpgart,nvidia,nvidiactl,nvidiafb}

Those init scripts do:
load the agpgart module
load the nvidia module
create the required device nodes in /dev
unload the nvidiafb module

Of course they should execute in proper order during boot
instead of the order that I listed them above.

The NVIDIA-Linux-x86 spell selects the correct source tarball to use
based upon the $HOSTTYPE variable.
That way it installs the x86_64 version on x86_64
and the ia32 version on ia32.
If your a checking the installation of it on a x86_64
then you may want to search within any case and if statements
involving $HOSTTYPE.
It might only be broken for x86_64
I have only tested it on ia32.
But I expected that it should work on both architectures.
