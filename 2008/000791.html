Your solution will go out in the next sorcery.

This is probably one of the least used methods for sharing a repository.
Other method are:

NFS, coda, sshfs, or ftpfs
to provide the remote repository mounted like a local filesystem
on a point such as /var/cache/sources.shared/
And having SOURCE_CACHE[1]=/var/cache/sources.shared
in /etc/sorcery/local.config

Another method is to have a samba server act as a caching web proxy.
That way when sources are downloaded by more than one box that
accesses the samba web proxy the sources can be transferred from
that box running samba instead of being downloaded across the Internet.

All methods are valid.
The fastest method is debatable.
However, the methoud using SOURCE_CACHE[1]
also requires the scavenge feature enabled.
And I have not tested it recently.
