On Monday 12 May 2008, Kyle Sallee wrote:
> Networking used to start in runlevels 2 3 4 5.
> However, networking will now start in sysinit before mount.
> Likewise NFS client support will now start in sysinit
> after networking and before mount.
> This way NFS filesystems can be mounted also.
> It should be possible to have Sorcerer boxes that have / on NFS.
> If a box network boots also
> then creating a Sorcerer thin client
> should be easy to accomplish.
>
> If starting networking in sysinit presents a problem please
> provided a detailed report.

I can see a potential problem for wireless networking because various 
utilities are needed for proper configuration. For example, wpa_supplicant is 
often needed and that in turn may need to run some local scripts. 
Furthermore, many wireless cards must load firmware from /lib/firmware when 
the proper kernel module is loaded. In short, I am afraid of this change.

BTW, what about ppp connections? Are those going to be affected in any way? I 
found recently how handy it is to have this functionality still available, 
especially when on travel.

> Most networking utilities are already installed in /sbin
> and therefore should be available on the initramfs.
> So far one potential problem exists.
> nmap will not be available from the initramfs.
> It is a c++ programs and requires libstdc++
> Therefore, please check activate networking templates
> and disable or replace the checks that use nmap.
