Power failures do not normally wreck Sorcerer boxes.
I would know.
Happens to my boxes.
2 of my UPS are beyond their battery life.

The sorcery tools execute sync after important events.
Therefore, if the filesystem has a properly high commit value
an unexpected power outage has a very low probability of
creating a situation where a spell is not installed that
should be installed or is partially installed and not tracked.

Sorcerer updates software faster than fast food places hire employees.
100 update per month would be low.
300 is not unreasonably for a deployed box.

If you prefer less updates then you can place the more frequently
updating spells on secure progress.
Then they will update only when the
new version is announced as a security fix.
However, a box can not be run with secure progress on all installed spells
indefinitely since library age in contrast to the software that updates will
eventually become too great.
When that happens compile failures will happen that are not normally expected.
A good example of that are boxes not surviving the coreutils update,
because either the installed linux kernel was too old or because the SA
never rebooted to the newer installed linux kernel.

I merely make updates available.
Selecting stable, secure, unstable progress and when to update boxes
and what software to update are decisions for SAs to make.
However with that in mind,
boxes that update every 2 weeks have fewer
update problems than boxes that update every 2 months.
