I can not allow uname -r

Let us assume linux-modular casts.
A newer version of linux installs.
Now NVIDIA-Linux-x86 casts.
Do you want it to install a modules for the linux version that is running?
Or do you want it to install modules for the linux version that will
run next boot?
Obviously, if NVIDIA-Linux-x86 uses uname -r
then xorg will not start on the next boot,
because there will be no kernel module
for the running version of linux.

If NVIDIA-Linux-x86 recasts after linux-modular casts
then that indicates NVIDIA-Linux-x86 is already installed.
If it is already installed
then the modules for the running version of linux
are probably also already installed.
When the new version of NVIDIA-Linux-x86 installs
the old NVIDIA-Linux-x86 kernel modules are not deleted.

Both /boot and /lib/modules are pruned by a method
that does not involve dispel.
That way if the newer linux does not work on boot
then booting to an older installed version of linux
aught to be possible.

I hope that explains why I must sedit all the uname -r
out of source trees?
