> "Kyle Sallee" <kyle.sallee at gmail.com> 
> If anyone wants to do testing of compression and decompression
> speeds using pbzip2 vs bzip2 on a dual core box and report results 
please
> that would give greater support for allowing pbzip2 to replace bzip2.

Tested, 1 core, 2 core and 4 cores, compression and decompression, with 
bzip2 and pbzip2, here are the results :)

File was 100MB created from urandom.
Processor used in test was:
Intel(R) Xeon(R) CPU           E5345  @ 2.33GHz
RAM size 512MB
Distro: CentOS 5.2 64-bit, Xen Dom0 v3.2.2

bzip2 compression speed:
real    0m26.921s
user    0m26.558s
sys     0m0.360s

bzip2 decompression speed:
real    0m11.113s
user    0m10.813s
sys     0m0.284s


And here are pbzip2 results:
1 core compression (using switch -p1):
real    0m27.496s
user    0m26.266s
sys     0m1.228s

1 core decompression:
pbzip2 1 core decompress
real    0m11.257s
user    0m11.081s
sys     0m0.136s

2 core compression (using switch -p2):
real    0m13.361s
user    0m26.526s
sys     0m0.096s

2 core decompression:
pbzip2 2 core decompress
real    0m7.286s
user    0m13.649s
sys     0m0.404s

4 core compression (using switch -p4):
pbzip2 4 core compress
real    0m8.582s
user    0m32.318s
sys     0m0.728s

4 core decompression:
real    0m4.115s
user    0m13.389s
sys     0m0.476s

AFAIK decompression speed also pretty good, uses all cores in there too.
