/etc/sorcery/log/etc/ is no longer used
/etc/sorcery/log/id/ is no longer used

/lib.old is now /lib.sav
/lib64.old is now /lib64.sav

/usr/lib.old is now /lib.old
/usr/lib64.old is now /lib.old

Explanation:
1. storing default config files wastes space
2. id logs are not essential,
  ELF logs contain enough and are faster to parse
  during dispel than ID logs.
3. Copies of /lib64 or /lib are not actually old,
just saves of the last copy expected to work.
4. Only one library cache is required.
If ELF 64 and ELF 32 are mixed on a box
then that box will have serious problems
since a library preload such as sentinel
can only works with the same type of ELF as sentinel is.

The above changes will make sorcery a little bit faster
while at the same time saving space in /etc
Also indexing speed is now about twice as fast
when generating a full index.
Blink and you may miss seeing it happen on an x86_64 dual core.
