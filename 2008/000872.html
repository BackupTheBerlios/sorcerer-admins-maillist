augur has changed somewhat:

source     none            List spells with non source override
source     spell none      Return to default sources for spell
source     spell sources   Specify URLs and source files for spell
sources    dir             Copy sources from dir into the source cache
sources    file            Copy file             into the source cache
src        none            View sources for all spells
src        spells          View sources for spells
stable     none            View the stable versions for installed spells
stable     spells          View the stable versions for spells

The new source command is similar to progress.
It can be used to override the sources for a spell to include also
download locations.
In some sense the augur source command replaces the long lost augur
prefer command
since it can be used to specify different download servers or a
desired download server.

# augur help source

When alternate sources or alternate download servers are desired
then this command can override the url and source specified in a spell.
For example if one wanted to change the source for bash to trash,
but maintain the same available version then that command would be:
# augur source bash ftp://ftp.gnu.org/pub/gnu/bash/trash-\$VERSION.tar.bz2

Multiple sources and multiple download servers may be specified just as
they can be specified in spells.
However, they must all be specified with a single invocation to
# augur source

When using the augur source command the $VERSION variable can be used
or alternately a specific version can be specified which would have a similar
effect to progress except that sorcery's reported installed version
for the spell
may not correctly report the version which is installed.
Therefore, use of \$VERSION when specifying source overrides is recommended.

The augur source command may be a convenience or an inconvenience if
set and forgot.
However, the way it is currently implemented it no additional indexing
time is required if
no spells have source overrides.

Therefore, there is only a slight performance cost if a SA elects to
use source override.
Without explicitly writing an evaluation table this change clearly has
more boons than banes.
However, when very precise control is required then creating custom
spells is still a good idea.
