gtk-doc has been a problem in the past.
Some versions of it would not compile.
In most cases gtk-doc is an optional requirement for compilation.
Several spells have gtk-doc off or optional.
However pygtk recently made gtk-doc a mandatory requirement
in order for pygtk to compile and install.
Is that the spell at the base of the gtk-doc problem?

docbook, hmmm?
Fix the spell if you can, please.
Although I wrote it, I did so in a rather odd fashion.
Much to my surprise nowhere
could I find documentation on how to install docbook.
However, one tarball or zipfile did have something like
an install script so I attempted to mimic what it did for
different versions of docbook XML and XSL so that they
could be installed and have their catalogs correctly adjusted.

Much time has elapsed since I wrote the spell.
It probably works as good as it ever did.
And in several cases it does work.
But perhaps not all.
Sometimes it is just a matter of having the correct
version of XML or XSL sheets installed.
Unfortunately, source trees never specify
in the README or INSTALL files
what versions of docbook XML or XSL is required.

I feel the same way about docbook.

because it requires docbook2man.
