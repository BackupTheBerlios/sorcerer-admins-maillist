Writing a rule to prevent a module from loading maybe possible.
However, init-scripts can also unload modules.
And modules listed in
/etc/modules.rm
are unloaded during boot.

NVIDIA-Linux-x86.d/init.d.on/nvidiafb

Above is an init-script that unloads the
nvidiafb module during boot so that pre-compiled
binary module provided by NVIDIA-Linux-x86 can
be loaded instead.
In that instance using an init-script is easiest.

There are only a few modules that can not be unloaded.
And those require udev rules to avoid.

Most modules do not use much RAM.
If the box has plenty and there is no module conflict
then it may be easiest to leave modules installed.

# lsmod | sort -k2rg | head

Those modules eat the most RAM.
