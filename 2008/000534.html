On Fri, Jun 20, 2008 at 10:25 AM, J Decker <d3ck0r at gmail.com> wrote:
> can there be an alias for available called avail ?

Sure.
But you might want to use a symbolic link instead?
For example:

# mkdir -p /usr/local/libexec/sorcery/augur.d
# ln -s /usr/libexec/sorcery/augur.d/available
/usr/local/libexec/sorcery/augur.d/avail

In the previous style which allowed me some slack
I probably had avail*) for the available command.
I feel very comfortable typing avail,
But I would probably type
# augur ava<tab>

Even though I want augur avail
for my own convenience I do not want
to start setting up distribution wide command aliases
so that eventually almost every command has
3 or 4 names.
It would confuse me.

tab completion works on augur commands,
and on cast and dispel for providing spell names.
The spell names provided to dispel are only those
spells which are "lone" which means they can be
removed safely.
command completion speed for augur cast and dispel is improved.
It should be very quick now.  :)
