Note this email is again about problems that occur
only only on J Decker's boxes and
not Sorcerer distribution wide.
This problem is caused by custom mount init-scripts
that J Decker uses.

The problem you report is not distribution wide.
It is probably due to custom mount scripts not properly
moving virtual filesystems.
/dev/pts is probably mounted.
It probably is mounted on the initramfs.
It probably was never moved to the real root filesystem.
See the init-scripts spell in the grimoire
for a proper working example of how to
mount and move mounted filesystems during boot.
