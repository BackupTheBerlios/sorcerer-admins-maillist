On Sat, Dec 6, 2008 at 2:28 PM, Mike Flanagan <gentoo.mike at gmail.com> wrote:
> Hi and good afternoon , as I am running augur update on my box I'm
> getting a lot of failed downloads .
>
> My question is this , what would be ideal settings to navigate around this ?

Glad that you asked.
First, please report the failures if they are due to incorrect spells.

To immediately work around the problem is a new augur command:

source     none            List spells with non source override
source     spell none      Return to default sources for spell
source     spell sources   Specify URLs and source files for spell

# augur help source

When alternate sources or alternate download servers are desired
then this command can override the url and source specified in a spell.
For example if one wanted to change the source for bash to trash,
but maintain the same available version then that command would be:
# augur source bash ftp://ftp.gnu.org/pub/gnu/bash/trash-\$VERSION.tar.bz2

Multiple sources and multiple download servers may be specified just as
they can be specified in spells.
However, they must all be specified with a single invocation to
# augur source


By using the augur source command
a SA can immediately tell sorcery to start downloading from
a server that the SA knows the source can be downloaded from.

If you want to download sources yourself
then there is another useful augur command...

sources    dir             Copy sources from dir into the source cache
sources    file            Copy file             into the source cache

# augur help sources

When sources are downloaded manually
or downloaded on another box after having used minileech
then those sources can be properly and automatically imported
into /var/cache/sources by using the sources command

By using the augur sources command a SA can ensure that a source
is placed where it can be found within /var/cache/sources/
so that spells can use it.
