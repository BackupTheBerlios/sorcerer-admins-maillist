While I do not understand the question
I do know that samba again installed libsmbclient.so
and not something like libsmbclient.so.0
Therefore, installed software that dynamically links with
libsmbclient.so.0 will be detected as ELF Failures and recompiled.
The fixed samba spell is already committed to svn.

Since thatl fix is semi-permanent
I did not cross reference the samba install log
to see if the adjustments were successful
the first time I fixed samba to compile with samba version 3.2.1
After I fixed the compile bug I did not expect an install bug to follow.
Sorry for the inconvenience, but it seems to be an issued
caused by the samba authors for not installing that library
with a proper name.

The recompilation of other spells might or might not be related
to the update of samba.
# augur event $SPELL
Will usually give an indication:

# augur event xine-lib
Cast Reason  4
Compile      3
Dispel       3
Download     2
ELF Failure  1

Thu Aug  7 09:10:26 MST 2008    Required ELFs Missing:
/usr/lib/libsmbclient.so.0

Thanks for the bug report.
