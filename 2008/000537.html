Kyle Sallee wrote:
> Evert Vorster found a bug that sneaked into the sorcery utils while
> simplifying the support for init-scripts.
> Therefore, I am sending out a newer sorcery with
> the newer modular augur a bit sooner than expected.
>
> Please be sure to change instances of
> # augur synchronize
> to
> # augur sync
>
> Also, check the augur help menu for command name changes.
> Some commands are removed, others enhanced, and a few changed names.
>
> Lastly, one of the advantages of modular augur is that if the module
> that a SA writes and installs in /usr/local/libexec/sorcery/augur.d/
> has the same name as one provided by the sorcery utils in
> /usr/libexec/sorcery/augur.d/
> then the SA installed augur command module will be run instead
> when that command is invoked.
> By this means SA can more simply customize augur functionality
> rather than writing functions into /etc/sorcery/local.config
> For example SAs could write their own command module for:
> # augur scry
> or any augur command.
>
> Thanks to Jean-Luc Malet suggestion
> the cast spell configuration menu
> is now a spell configuration checklist.
>
> Thanks for the bug reports and feedback!
