Jan Merka discovered that /usr/libexec/sorcery/sentinel
is not installed as part of the installation process on the new I/Rs.
This is because /usr/libexec/sorcery/sentinel is an alien.
It is installed by sorcery-sentinel, but not tracked as installed.
The sorcery-sentinel spell has been fixed.
However, in order to use the new Sorcerer installations:
# cp /usr/libexec/sorcery/sentinel-$HOSTTYPE /usr/libexec/sorcery/sentinel

Sorry for the inconvenience.
It will be at least a week before new fixed I/Rs are available.
