Previously, cast used CPU cycles each during each spell
to determine if depmod should be run
and to determine if symbolic links in /etc/rc.d/*/ should be regenerated.

Now, modules.dep and symbolic links to init-scripts
are regenerated as required during shutdown.
This is very much like initramfs where a check is executed
first during shutdown to determine if regeneration is required.

Will it make shutdown slower?
Both the checks together cost less than a second.
However, if updating is required
then shutdown time will be a few seconds longer.

Overall this minor and almost inconsequential change
conserves CPU during each spell cast.
However, please remember to issue
# shutdown -r now
or
# shutdown -h now
after extensive updates and especially when a new kernel is installed.

If you want to run the init-scripts manually
then the order for execution is:

/etc/init.d/depmod
/etc/init.d/rc.d
/etc/init.d/initramfs

parameters are not required.

Starting with sorcery implosion-20080616-0
cast will no longer run /lib/lsb/install_initd nor depmod.

The above described change is
transparent and inconsequential to most Sorcerer SAs,
however I expect a few SAs will want to know about it.
