>
> It touched upon an important issue that I wrote about on the list
> previously.
> If an x86_64 installation has an ia32 installation in a subdirectory and
> the
> ia32 installation is maintained then the two should be possible to
> staple together.
>
> Let us assume that the IA32 installation is /opt/ia32
> First, /lib/ld-linux.so.2 must be a symbolic link to
> /opt/ia32/lib/ld-linux.so.2
> Second, /etc/ld.so.conf must contain lines:
> /opt/ia32/lib
> /opt/ia32/usr/lib
> Finally, ldconfig must be run.
>
> That aught to work, but I am awaiting for anyone to report success trying
> it.
> I eventually plan to try it in qemu x86_64 with separate disk image for
> an ia32 sorcerer install and x86_64 install.
> However, at the moment qemu-x86_64 is busy compiling updates.
>
> An IA32 installation stapled onto an x86_64 installation
> should allow running pre-compiled IA32 executables such as AdobeReader,
> flash, and even IA32 wine for running win32 software.
>
> Due to the limitations of libtool, pkgconfig, sorcery-sentinel, and sources
> that install into lib and not lib64 directories on x86_64 architecture
> stapling separate installations of x86_64 and ia32 Sorcerer together
> should be the most sane and safe way to acquire the functionality.
>
