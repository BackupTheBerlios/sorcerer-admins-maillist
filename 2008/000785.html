Anyway, I wrote before:

# chmod a-x

Difficult to mistype.
However, when I set modes I tend to type:
chmod 0600 /etc/init.d/some-script
That way I know the mode is exactly what I want.
Or chmod 0700 /etc/init.d/some-script if I want it executable.
Remember when it comes to root
any executable bit set on a file makes it an executable,
but mode rules apply as expected for regular users.
However, init-scripts are run by root.
Therefore, 600 and 700 are the best choices for setting the mode.
Since users and others will never be reading, writing, nor running
init-scripts their modes are 0 for no absolutely no permission.
