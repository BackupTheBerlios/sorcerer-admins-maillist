On Thu, Aug 28, 2008 at 2:57 PM, J Decker <d3ck0r at gmail.com> wrote:

> I LIKE the idea about using uuid for disk partitions and lilo boot
> specifications.
>
> But, this doesn't work for the purposes of taking an existing disk and
> copying it to another disk.

It can work.
The UUID for a devices is can be changed.
However, it probably should not be.
The purpose of UUID is that they should be unique.

> 1) Lilo complains that there are two disks with the same ID - long
> time issue which used to be fatal.

Change the UUID of the copy before running lilo


> So, I guess, I dunno I have to hand go back through and replace UUID
> references with device references... after installation Sorcerer
> doesn't really manage fstab and lilo.conf.head right

That is the idea behind using the UUID.
Find it and use it forever.
Edit /etc/fstab and /etc/lilo.conf{,.head}
to use device names or UUID or labels.
I have no preference.
However, the I/R uses UUIID so that installations
can be made onto both fixed disks and
also flash drives.
But none of my deployed boxes,
except flash drives, use UUID.

If the device names change UUID continues to work.
It does not matter if /dev/sda1 becomes /dev/sdb1
or if /dev/sda1 becomes /dev/hda1
With UUID the proper device with
UUID will be mounted.

There is no ideal method.
Each of the three has different boons and banes.
