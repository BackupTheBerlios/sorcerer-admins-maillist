2     LA Failure     lftp         -      00

The LA Failure on lftp should be standard on every box with lftp installed.
LA Failure indicates something wrong was detected in the installed
spells's libtool archive file.
cast corrects this by recasting the installed spell.

In the case of lftp, this is a false positive.
It is caused by an incorrect method sorcery
uses to construct LA logs while still being able
to detect existing irregularities in installed libtool archive files.
However, the correct method is used when lftp is cast
and therefore lftp does not continue to show up as an LA Failure.

This functionality is in response to the reports that
libtool is not properly adjusting to changes in the location of
installed libraries.

The new libtool archive check combined with the ELF check should
finish our concerns about ELF library installation and updating.
Fortunately, pkgconfig uses an entirely different method than libtool
and therefore does not suffer any problems similar to libtool
when the installed location of libraries changes.

If any libtool related problems persist after the update to
sorcery version implosion-20080510-0 from
grimoire 2008-132 then please report them.
In theory if a malformed libtool archive file is created
it could cause a spell to recast repeatedly.
However, I have not seen it loop, yet.
