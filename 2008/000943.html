If it turns out that Python 3.0 is required
then it can be added as a spell that installs into /opt
For the moment I anticipate that might cause problems.
Perhaps we should instead wait and see what to do about it later.
Whether Python 3.0 is backward compatible or not with 2.x
is not a pressing issue for Sorcerer since sorcery is in BASH
If SAs want Python 3.0 they can either select it or write
a custom spell for it and install it where they like.
Although Python is a scripting language I consider it to
be in the same classification of updates as libraries are.
Basically it is support software.
After software that requires Python catches up to the Python 3.0 release
then Python can be pumped up to Python 3.0
and an older Python-v2.... can be added like we did in the past.
