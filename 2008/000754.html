Logs important for sorcery to understand
the state of installed software
are still kept in /etc/sorcery/log/
However, logs that can be removed
without consequence are now stored in /var/log/sorcery/

Removing logs in /var/log/sorcery/ may cause some loss of information
such as events, activity, download logs, etc...
This information is rarely something that the sorcery tools use.
However, the event logs are sometimes used by sorcery.

If logs in /var/log/sorcery are accidentally cleaned by a log cleaning program
then that should not cause a problem for sorcery if those files are removed.
The point is if some space is needed quick
then logs in /var/log/sorcery/ are safe to remove.
