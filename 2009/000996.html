kde-v4 has a file indexer called stringi
I do not know if it is useful yet.
However, in order to use it,
it requires qt-x11-opensource's mysql plugin.
Therefore, it requires mysql.

I changed the mysql spell to make it work better
and to also always create mysqld
and to start it as user mysql using an init-script.
Also I modified qt-x11-opensource to require mysql.
Therefore, mysql will be installing on any box with kde-v4 installed.
However, the stringing might still not become available until
qt-x11-opensource recompiles.
Recompilation of qt-x11-opensource will not be forced.

Apparently, my initial conjectures that mysqld must be running
as user mysql were incorrect.
Apparently, some part of the user's kde session starts mysqld
with a custom configuration file.
Makes good sense that way.

I did not decide exactly what I wanted to do with mysql.
So mysql shoud not be run as user mysql at boot
then # chmod 644 /etc/init.d/mysqld
Otherwise, please run mysql_secure_installation to secure the installation.

mysql compiles, installs, and runs.
stringi started running without complaint for my user account.
It indexes file in my home directory.
It seems to do that frequently.
