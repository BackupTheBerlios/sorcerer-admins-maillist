On Sat, May 9, 2009 at 9:38 AM, Evert Vorster <evorster at gmail.com> wrote:
> Hi there...
>
> Am I right in assuming that statically linked programs can also never suffer
> from ELF failure?

An executable can be linked with several libraries.
Some of those libraries can be ELF, some can be static.
The content of static libraries becomes part of the executable.
Therefore, if an executable is entirely static linked
then it ca not suffer an ELF failure.
An entirely static linked executable would also not depend upon
ld.so nor libc.so.6 in order to run.
Even if /lib/ or /lib64/ were missing the program would run if invoked.
