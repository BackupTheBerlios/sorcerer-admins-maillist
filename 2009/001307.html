Opps.
That box is messed up.
Hope you have a complete older copy of the udev
installation in /etc in order to copy out into the initramfs /
and run.
Because if an older version of udev was compiled
with the current udev build() or a newer version of
udev was compiled with the older udev build()
then it will probably be unusable.
You can blame the udev authors for changing
the build method, but also thank them for
documenting the current proper build method
in the INSTALL or README file.

I tested the udev with the udev spell
and rebooted the box to be satisfied that it works.

If I were in your situation then I would probably
boot with parameter rdinit=/bin/bash
Then after landing at a shell prompt I would
manually load modules.
Then I would run:
# /etc/init.d/dev start
# /etc/init.d/dev-cp start
After that I would start creating device nodes
for the disks and partitions.
After it looks good I would do:
exec /init
And hopefully the box would boot almost normally.

After the real filesystems are mounted and the box
is in it's normal runlevel I would make certain that
udev version progress is stable.
Because my guess is that it is set to latest
or a specific version.
Therefore,
# augur progress udev stable
Then,
# cast udev

After that the box can be shutdown normally
which will cause the initramfs images to regenerate
with the fixed udev and the box should be able to
reboot normally.

Sorry for the inconvenience.
