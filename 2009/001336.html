xextproto is verison 7.0.3
and that libXext is version 1.0.4
However I just completed a cast of libXext that worked.

that I would recreate
and see the error that you saw.
But at the moment it eludes me.

Are you absolutely certain about the version
progresses that you have set and forced for
software installed on that box?

By executing:
# augur progress

it will remind the SA about what version progress is set.
However, if certain version progress was changed,
but a newer and update was not completed after
then unexpected previously selected versions
might be installed instead of having it all congruent.

Most of the versions listed as the most current version
in stable are tested once known to work and usually
continue to work.
However, versions listed in the unstable progress
are not tested and probably will not work.
Some might work if all the appropriate spells
are forced to unstable version progress.
However, since xorg related spells often have
different source trees that install headers
and different source trees that install libraries
it is usually a less than trivial task to use
unstable versions of xorg related spells.

However, since this is Sorcerer the SA
does receive the final say about what is installed.
But if any of it are from non stable version progress
then I probably will not know what to expect or why
something went wrong.

So far my best guess is
the most obvious and easiest way to make it fail.
Perhaps a version progress is being forced?
Please double check.
