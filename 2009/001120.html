There is nothing odd looking about monotone.ca
It has a single a record
and it has 6 DNS servers that are authoritative for it.
All the DNS servers are owned by easydns.com, apparently.
If the lookup was failing could it be due to the DNS caching server
that your box connected with?
However, if your box runs it's own DNS server
and consults that instead
then a DNS resolution of monotone.ca should not be blocked.
However, it is possible that it was not possible to reach those
authoritative DNS servers for the monotone.ca domain.

If you want to see what I am seeing, type:
$ dig monotone.ca a

dig is provided with bind.
bind is a DNS server.
