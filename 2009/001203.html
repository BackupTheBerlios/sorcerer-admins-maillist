I am currently testing candidates for the next IA32 I/R
However, after 3 days I still do not have a winner.
The most recent problem is:

* rpc.idmapd starting
/sbin/rpc.idmapd: error while loading shared libraries:
libevent-1.4.so.2: cannot open shared object file: No such file or
directory

That is because I had it installed in /usr/lib/ instead of /lib/

Fixing that now.

When I have a winner you can be the first to download and try it
if you want.
I will post the announcement on the email list when it is ready.


The problem I overcame previous to the one listed above
was that tar now automatically escapes filenames that contain spaces.
Consequently, tar had an error when creating atk and ustr archives
and the sorcery tools removed those archives,
and thus the previous I/R was missing those two archives,
which caused attempted leeching and ELF failures during
the compilation that happens during installation.
And so that problem was fixed already.

So basically, I must keep hacking at the bugs
until it is perfect.
Please wait a while longer.
