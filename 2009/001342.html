The xorg site website always seems to look
as if the next big release is imminent.

Probably the way that it is now encourages faster
and further development of the individual components,
but makes integrating the xorg components more difficult.
I suppose it is possible that the XFree86 group
expect this would happen if the source was fragmented
into a zillion different tarballs.

For now we must wait to see if someone sows it together.
For the time being sitting back a bit from the edge of
xorg development seems as if it might offer the best
compromise between bug fixes and stability.
