Fixing audio?

# echo "options snd-pcsp index=31" >> /etc/modprobe.conf
# depmod -a

Then, unload and reload all sound modules.
Or it will work as expected on the next boot.

Someone already committed changes to madwifi in svn
That might make it work?

You can reboot to linux-modular 2.6.28.3 simply by editing
/etc/lilo.conf* and then rebooting.
Trying to recompiling to it might not have expected results.
In the grimoire are only the configs for 2.6.28.5 for linux-modular.

With the kernel configuration as it is now the PC speaker can emulate
a sound card.
However, it sounds very scratchy and volume can not be adjusted.
But frequency sounds about right.

There are a few different methods for selecting which has priority
when it comes to alsa audio.
There are docs about it in /usr/src/linux/Documentation

Also the audio device can be selected using alsaconf

Also with KDE4 the a different audio output device can be selected
from the user's configuration.
K menu -> System Settings -> Multimedia

The alsa/kernel developers are responsible for the decision
to make the pc speaker load as the first audio device.
It probably is not what most people want.
