>> 2) in sorcerer it's not possible to use software raid on top of lvm,

>> not sure that raid array can be dynimically updated like lvm allow
>
> Why not using the software raid and create LVM on top of that? I have been
> running such configurations on half a dozen machines for about two years
> without problems.
>
> J.
well for several reasons :
1) raid 0 is too dangerous
2) raid 1, raid 0+1 are costly and my use don't require this kind of safeness
3) raid 5 and 6 are dangerous (less for raid6) and recover from an
hardware failure is long process during wich it is likely to loose
another drive and then all data
4) raid 3 is interesting but is dangerous as well, and it seems that
one of the best solution is what I would call raid3+1 : raid 3 with 2
checksum drives in mirror, however this solution isn't available
neither in hardware nor software and can be achieved with a
software+hardware combinaison.
5) I don't have multiple disk for doing raid :), however I was
thinking about this using flash storage....
anyway this was just some question/thoughts
