Be careful installing LVM2 version 2.02.52, it creates a big problem because 
it installs its own libdevmapper.so.1.02 library into /usr/lib while device-
mapper already provides a library of the same name and version in /lib. As a 
result, executables like /sbin/vgscan and others will stop running and a 
message about a relocation error in libdevmapper.so.1.02 will be issued. That 
is if you are lucky to try it before rebooting. 

If you try to reboot first instead, and your root file system is on LVM, grab 
your rescue CD and have fun... You have been warned.

I will investigate a proper solution when (if) I have some time but that's not 
going to happen in the next few days. Sorry. 

For now, I am sticking with LVM2 version 2.02.39.
