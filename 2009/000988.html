personaly I don't like modifying the mbr and for various reasons like 
having muliple IDEPENDANTS Os (a sorcerer x86_64, a i586 version and a 
widows) I don't want to have it installed on mbr
shoot the mbr and you shoot everything,
shoot the bootloader and you can't even switch to another os without 
trouble in case the bootloader is in first sector of partition just 
change the active partition and it'll work just fine....
if you have several os installed then all are running race condition 
about whom will write the mbr..... either you have to maintain all the 
configuration of all os to be certains they will keep track of the 
other, or just install the bootloader on 1rst sector and let they live 
their lives.... I can even update a i586 sorcerer from the x86_64 one 
without bothering ;)
anyway if grub has to be installed into mbr I will likely never use it....
look at the "rescue usb disk" I made, without lilo it wouldn't even 
exist or you will have to waste a full usb drive for having a small 
rescue disk, on this you have a lilo installed on boot sector of the 
partition, allowing to move it anywhere you want... on a sdcard, on an 
usb drive, on flash, on a harddrive.... it's STANDALONE

I think that lilo shall use 1rst partition block as default and optional 
mbr (for crapy filesystems that don't support having a boot sector) and 
shall use UUID by default (and not the old /dev/sd* or /dev/hd* or 
/dev/mmc* or.....) like this anywhere you go, whatever the physical 
medium of your drive, it will boot as expected

Kyle Sallee wrote:
> I have nothing against using grub.
> Someone who uses grub should write the support for it.
> I use lilo.
> That is why the initramfs support and integration with lilo is excellent.
> Unlike grub that must be installed in the MBR
> lilo can be installed both in boot blocks or the MBR.
>
> Edit /etc/lilo.conf*
> Please remember that /etc/lilo.conf and /etc/lilo.conf.fast
> are constructed files and therefore to properly make the change
> also /etc/lilo.conf.head must be adjusted.
> Chance the boot = to specify the entire disk instead of a partition.
> So instead of /dev/sda1 use /dev/sda
> Instead of /dev/hda1 use /dev/hda
> Run lilo
> Look for error messages.
> Reboot.
> Enter BIOS/CMOS setup menu
> Make certain that disk is on the boot list.
> Continue booting.
>
> It is true that I do not present an option about where to install lilo
> in the menu driven installer.
> It selects the boot block of /boot if /boot is definied
> or the boot block of /
> However, that should be sufficient for installation.
> After the first reboot just use the above steps.
> Two lilos will be installed.
> However, since booting runs the MBR first
> the lilo in the MBR will run.
> In order to return to using the lilo in a boot block
> a ms-sys bootable MBR must be installed in the MBR.
