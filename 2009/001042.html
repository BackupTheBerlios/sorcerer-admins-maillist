Either use the menu driven installer.
Or do a manual installation when certain
of how to set up lilo's configuration files
and /etc/fstab correctly, and to use the init-scripts
to regenerate symbolic links in /etc/rc.d/
regenerate initramfs images,
and install lilo into the boot block or MBR.
It is actually rather straight forward.
All the software required for a base system
is available on the CDROM in /var/cache/archive/

However, between you and me,
I would probably use the menu driven installer,
especially if I wanted a setup involving encryption.

By the time a login prompt is presented,
when booted into the intiramfs on the Sorcerer I/R,
the Sorcerer I/R is mounted on /media/IR/
The Install/Rescue CD only has a gzip compressed initramfs,
a compressed linux kernel, and a lot of 7za compressed tarballs
in /var/cache/{archive,source}/

The apparently booted filesystem of the I/R is only an initramfs image.
Do not expect to find anything in /var/cache/{archive,sources}
The fun files are in /media/IR/var/cache/{archive,sources}

For some odd reason I expected that the I/R when booted should display.

Welcome to Sorcerer.
Please login as root.
Press enter for password.
/sbin/sinstall
to begin a menu driven installation.

That is really all the install instructions.
Does it not print?

Perhaps, the confusion is because I changed
it to sinstall instead of keeping it sorcerer.install.
I have no idea why I changed it.
Ah, maybe because sorcerer.install was long to type?

Please remember not to select optionals for spells
where the optional is not already installed
while booted from the I/R.
The network will not be up.
The optional source tarballs will not be available.
And installation will not proceed as smoothly as expected.
That potential problem is fixed in the next Sorcerer I/R release.
