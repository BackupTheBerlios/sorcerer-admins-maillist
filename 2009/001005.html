Hi agree with evert,
since I don't install often sorcerer (to be franck the last time I did 
was due to a total mess of the FS after an upgrade, and to be franck 
it's so long that I don't remember when hahaha) I rarely contribute on 
feedback about the installer. But if we want to have a good installer we 
shall do it often
"I do not agree with having the option to select any spell for which you 
do not have sources until Sorcerer is installed and network aware." I 
also agrees about that
for my point of view I thinks that an I/R shall have everything to run a 
console and to connect to some network, it shall be a standalone 
sorcerer, simply it boots from CD instead of harddrive
after some testing with usb drive I noticed that one of the fastest way 
to create a bootable device is simply to copy from an existing sorcerer 
(either cdrom or harddrive) and to configure lilo and fstab to use 
/dev/disk/by-uuid/* entries instead of legacy /dev/sd* or /dev/hd* or 
anything else.... this is really the fastest way and the most secure 
one, it can be performed without having to boot on the I/R (ie from an 
existing sorcerer, from any linux, even from windows, and I even created 
it using a virtual machine under windows (don't shoot the messenger ;) 
)), and this has several advantages

so from my point of view, what an I/R media be is : a network aware 
minimalistic (console only) sorcerer box that can copy itself to another 
media through a menu

an additional interesting point : I don't aggree with evert about the 
skillcheck when trying anything different with the install : if a 
modular kernel (ie a kernel that has every module compiled) wasn't 
provided it would be hopeless to even boot the rescue cd from an usb 
drive or a virtualised host.... what I see is that the issues are not to 
boot the I/R and to run it in any env, the issues arrise AFTER the 
install! for some reason, after the install, the state of the box isn't 
the same than while booting the I/R
you can have everything running on the I/R and can't even boot on the 
new install....

best regards
JLM
