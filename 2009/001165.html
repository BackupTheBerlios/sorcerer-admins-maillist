I am concerned that libcap is no longer usable or is obsolete.
Once the problems are over with we can add it back in.
However, it is entirely optional.
It can only be used if it is detected.
glibc's current() removes the header file
that tends to cause it to be detected.
And libcap will dispel if installed.
All boxes should on update be dispelling libcap
and also eliminating the possibility of libcap related errors.

I do not expect that any Sorcerer box uses capabilities.
It must be enabled nearly from boot usually using a patched /sbin/init
There should be no problem with eliminating libcap
and thus avoiding any compile errors caused by it.
