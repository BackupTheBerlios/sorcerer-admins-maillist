sourceforge again has reorganized the way downloads are made available.
The good news is that final URLs are not random and the URL obfuscation is weak.
Therefore, files can still be downloaded using a single request,
and avoid having to do the dual URL redirection that sourceforge
authors want for people to do while downloading from the sourceforge
website using a web browser.

The bad news is that there is no definite expected URL pattern uniform
to all projects.
This translates into roughly 60 hours of work
which will be required to manually check and correct URLs
for all spells that have sources acquired from sourceforge mirrors.
If I can mass acquire all the information required to make the changes
from a sourceforge rsync server
then I may be able to cut the workload down to 30 hours.

If you can not download from sourceforge for a while
then you already know who to complain to for changing the way
files are downloaded from sourceforge.

First, I rewrote the SOURCEFORE_URL mirror file.
