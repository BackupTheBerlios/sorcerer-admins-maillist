I expected that it was a udev bug that they might someday fix.
I have not seen the warning message print as an endless loop.
I have seen it print a few times.
I ignore it.

Please send a fix for it if you figure it out.
Beware that a mistake involving the udev
will render a box unbootable by normal means.
In such a situation boot using kernel parameter
rdinit=/bin/bash
then repair the problem using the udev that you
saved previously somewhere in /etc/
so it would go onto the initramfs.

As far as bad breaks go...
Having udev messed up is about as bad as
having glibc, ncurses, readline messed up.
The problem can be repaired without booting from I/R
if the SA knows what to do.
Otherwise turn up the AC,
because the SA is about to break a sweat.

For a while I have not considered
this annoyance of a warning or error message
worth the risk of breaking udev.
Afterall, rmdir //dev seems like a command that
would or rather should never be issued.
