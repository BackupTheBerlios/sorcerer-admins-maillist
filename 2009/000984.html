Hi all,
I created using the latest rescue disk a "preview" of usb recover disk
********THIS IS  TOTALY UNOFFICIAL ******************
** made this on my own initiative **

location : 
ftp://cityhunter.is-a-geek.org/public/linux/sorcerer-ia32-20090101.usb
content : last rescue disk, bootable ext2 partition

installation notes : (not tested yet but what will be)

create a partition on a usb drive of at least 519Mo,
mark it as active,
ensure that a bootable mbr is on it (on windows it shall be created on 
partition creation, on linux use ms-sys -s <the device>)

on linux :
dd if=/path/to/the/image/sorcerer-ia32-20090101.usb of=/dev/thenewpartition

on windows : it require rawrite 0.9 at least (or any rawrite with 
support of disks and usbkeys)
use the GUI to install on the right drive

reboot, plug the key, and enjoy

known limitations/bugs
- initramfs is updating at each reboot... don't know why, so fast won't 
be displayed :(
- on boot the keyboad layout is displayed, you have to wait a litte here 
so that the usb subsystem is loaded and /dev/entries are created
- when doing a reboot the kernel is reloaded by a kexec call.... so the 
bios is never displayed again (really annoying when you want to go back 
to normal boot)

best Regards,
JLM
