Hi.
I added the spell xorg-legacy
It switches the version progress to legacy progress
for spells with the X11R7 attribute and that also have a legacy version defined.
And I also added many legacy versions to spells with the X11R7 attribute.

Basically, I took a look at what versions I was using on my workstation
and set those for legacy versions in the grimoire.
I am hoping that if it works on my workstation
then it will work on your box.
My workstation is using the NVIDIA-Linux-x86 pre-compiled proprietary driver.
Therefore, casting xorg-legacy and doing
# augur newer; augur update
after it will probably install a set of X11R7 spells that are suitable for use
with NVIDIA-Linux-x86

Recommendations for legacy versions are welcome.

For those lagging back behind the terrifying update,
this might provide an easy method to get back on track.
Or I might be off by a few spells and it might still require some work.
Those who have been updating have already encountered
the new kde-v4 release and new gnome releases,
and of course other new releases of fun and useful software.

Please lend your assistance to the proper addition of legacy versions
to X11R7 spells so that boxes can move forward while avoiding a
hazard of using X11R7 sources that are too new to be used with
proprietary pre-compiled binary drivers.

Glad, I created that legacy version progress.
I knew it would become useful someday.  :)
As for the xorg-legacy spell.
If new legacy versions are added for spells with X11R7 attribute
then only the xorg-legacy spell's stable version should be modified
in order to effect the change.
The xorg-legacy spell does not have anything recorded about
which X11R7 spells have legacy or which spells are X11R7 spells.
The xorg-legacy spell gathers that information from the index.
Therefore, completing support for legacy versions of X11R7 spells
aught not to be an ordeal.

Lastly, the xorg-legacy spell is not a substitute for the xorg spell.
It is not that type of profile.
The xorg-legacy spell does not require any spells.
It merely adjusts a box's selected
version progress for spells with the X11R7 attribute.
The changes made by xorg-legacy,
which only effect files in /etc/sorcery/log/progress/
is probably undone automatically
when xorg-legacy is dispelled.
The effected spells should  go back to
stable version progress after xorg-legacy is dispelled.
