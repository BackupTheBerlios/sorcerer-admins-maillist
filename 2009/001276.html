So gentoo's /boot directory is set up the same as Sorcerer?
They also use "initrf" as the same name for the initramfs image?
I wonder what the odds of that happening could be?
When I designed Sorcerer's initramfs
I did not copy any other distro?

Well, I suppose a good method for
creating an initramfs image is obvious anyway.
Yet I expected distros would use an initramfs
that was either included or attached to the kernel image
since that method of using an initrmafs is support by the linux kernel.

Then again Sorcerer is probably the model
for how source based distributions should work?
An initramfs combined with a kernel image is not sufficiently flexible
for a distribution that can change anything or everything with an update.
I do not mean to be bragging, but I am thankful that what we have works.

So now that we know that 75% or more of the initramfs
is consumed by kernel modules
then what does anyone want to do about that, please?
