I will start by saying that NVIDIA's method for installation is not proper.
The only method that properly installs and tracks files
is having two separate spells.
Therefore, there are now two spells for installing NVIDIA drivers from NVIDIA.

NVIDIA-Linux-x86_64 installs ELF 64-bit libraries and kernel module.
NVIDIA-Linux-x86 installs ELF 32-bit libraries.
On i686 boxes it also installs the kernel module.

Most boxes will have only one of the above two spells installed.
Multilib boxes will have both spells installed when OpenGL
is desired for running 32-bit ELFs or win32 applications in wine.

I know about the multilib file for download.
Unfortunately, it does not entirely track when installed,
because it installs some files directly into /usr.
Not only does it create annoying alien files,
but it corrupts files installed by MesaLib.

I also tried a single spell that uses each of the two single
architecture binaries.
That nearly works.
Unfortunately, NVIDIA's ELF 64-bit installer removes files from the DESTDIR
that were installed by NVIDIA's ELF 32-bit installer.
And so the installation falls short of becoming usable for ELF 32-bit OpenGL.

However, since NVIDIA provide the two single architecture files for download
having two separates spells is not entirely illogical.
Sorry for the inconvenience.
Please redirect all flame email to NVIDIA.

Hopefully, after the dust settles boxes can have either or both ELF
32/64-bit NVIDIA drivers
a kernel module, no aliens, and no overwritten MesaLib files.
