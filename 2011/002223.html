Ah, gcc optimizations. I have used -O3 and always there is some things 
that go broke. 
Maybe you do not notice it right a away but years ago when i played cflags 
i learned that do not bother to use -O3 because it brokes more than it 
make things go faster. 
I havent study newer GCC versions if there has been any improvements but i 
think -march=native is good option (or you could also use -mtune=native) 
if you are doing local builds and also -fpmath=sse (on 32-bit compiler 
this needs also -msse or -msse2) should be usefull option. 

I vote for -O2 anytime, -Os is also not usefull anymore. I think there was 
also a message that Linux kernel removed -Os from its flags because it did 
not made sense anymore with current CPUs. There was some test runs that 
showed -O2 and -Os produced about same amount cache misses, but O2 was 
about 8-10 % faster (on some one benchmark)  than Os...In that light O2 
wins by default.
