> Thanks for all the hard work, Kyle.

Thanks for the appreciation.
There is much to appreciate about these new I/Rs.
They have an additional revision of the Sorcerer init improvements.
And several init-scripts were modified in order to tidy output
that were not necessarily bugs but could attract a SA's attention.

I do not know for certain if the previous month's I/Rs
provided /etc/bind/named.conf and /etc/ssh/sshd_config
I made certain that these I/Rs install them.
That way sshd will start on the first non-IR boot
and DNS names will resolve.
The default sshd config does not allow root login.
So not having the root password set immediately is not a problem.
However, I still strongly recommend setting the root password
before configuring a network device.

Yanking bugs out by their wriggling tails is a good idea.
Errors involving segmentation faults
might be resolved by trying newer or older versions of the library.
Sometimes less aggressive compiler optimization flags fix the problem.

then please report that for a distribution wide fix.
