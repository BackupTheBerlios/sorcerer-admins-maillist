>> I require more information about the network.
>> Please specify the type of network one is connecting with,
>> whether it is 802.11, whether it is encrypted and any
>> other useful information.
>> Chances are that one can set up wireless using iwconfig
>> and then be assigned an Internet address via dhcp
>> by running dhcpcd followed by the interface name
>> such as wlan0 or whatever the interface name is called.
>> 
>> Both iwconfig and dhcpcd have manual pages.
>> However, for more detailed instructions
>> please repost the request to the email list
>> along with the requested information.
>> 
>> Many SAs are better at configuring wireless networking than I am.
>> Jan Merka would be better to ask than me.
>> 
>> SAs that want to permanently configure networking on a box
>> usually start in /etc/networks.d/templates/
>> and they read the README file.
>> Provided that the SA know some information about the network
>> then the templates method can be used to configure networking
>> without having to know about iwconfig, ip, and dhcpcd and
>> the command lines to type.
>> 
>> If it was my laptop and I just wanted to connect
>> it to the network for a trial and a test
>> then I would use iwconfig and then either dhcpcd or ip
>> 
>> It is easiest to think of it in two parts.
>> iwconfig tells the wireless adapter which essid, or network name,
>> that the wireless adapter should connect with.
>> Then dhcpcd or ip are the tools used to configure the IPv4 or IPv6 networking.
>> 
>> Please read the system log or the kernel ring buffer
>> to receive information about the success or failure
>> of connecting with the wireless network.
>> dmesg will display the kernel ring buffer.
>> The system log is stored somewhere in /var/log/
>> It might be /var/log/messages
>> However, the filename depends upon which syslogger is used
>> and how it is configured.
>> 
>> Usually, udev will have already loaded the module for wireless networking.
>> It might work, it might not, it might require some firmware.
>> A few spells for firmware are included in the grimoire.
>> Some of the come pre-installed.
>> If the wireless adapter requires firmware and
>> the firmware is not installed
>> then the wireless adapter will not work.
>> That is usually evident in the system log and kernel ring buffer.
>> 
>> # /sbin/ip addr show | grep "^[0-9]:" | cut -d : -f2
>> 
>> That lists the device names for network adapters.
>> The output will look something like:
>> 
>> lo
>> eth0
>> wlan0
>> 
>> eth are usually ethernet adapters, but might also be wireless.
>> wlan are usually wireless adapters.
>> 
>> udev handles the mapping of device to device names.
>> And it remembers.
>> So if a new wireless adapter is installed
>> then the new adapter might receive a name such as:
>> wlan1 or wlan2 or wlan3.
>> wlan0 will always belong to the first one that was found.
>> That way the names for adapters does not change
>> even if installed hardware is modified.
>> 
>> So as a summary.
>> First, turn on the wireless adapter and tune it to a network using essid.
>> Next, configure the networking using dhcpcd or ip.
>> Third, be certain of /etc/resolv.conf
>> name to IP address resolution does not work
>> when the contents of /etc/resolv.conf is incorrect.
>> And yes there is a manual page for resolv.conf
>> Then, try to load a webpage using lynx or links or curl.
>> Or if ping is available try pinging somewhere.
>> 
>> Lastly, if it does not work
>> then try posting on the email list or perhaps asking Jan.
>> 
>> If firmware is lacking
>> then connect the box to the router using a category 5 cable,
>> configure networking using dhcpcd
>> and download the firmware.
>> Install firmware files in /lib/firmware/
>> Then unload the module for the wireless adapter and reload it.
>> That will cause not only the module to load but also the firmware.
>> 
>> Every year I ask Sorcerer SAs about having some sort of menu driven
>> method for configuring networking.
>> And the replies are either that it is inflexible
>> or that the current method is fine,
>> or that networking only has to be configured once per box
>> so why bother having Sorcerer SAs learn something different.
>> 
>> So basically, we have the templates method.
>> But if that is not preferred then the SA can set up
>> networking however the SA wants.
>> The only wrong way to configure networking on a Sorcerer box
>> is a way in which networking fails to work.
>> If it works then that it is good to go.
>> 
>> My opinion is that the most bothersome or annoying part
>> about configuring networking is that so many drivers require firmware.
>> A freshly installed box probably has drivers but might lack firmware.
>> If the firmware is not installed and a spell does not exist for it
>> then the SA must first do a wired connection and acquire the firmware.
>> But even so that is better than the past.
>> In the past linux lacked support for many networking devices,
>> especially support for wireless USB networking devices was lacking.
>> But more devices are supported now.
>> Unfortunately, they tend to require firmware.
>> 
>> Provided the module, and firmware are available
>> then most 802.11 wireless adapters can be configured
>> using iwconfig and then either dhcpcd or ip
>> So all the tools are there.
>> The challenging part is discovering the command line parameters
>> in order to make them work as expected.
>> Unless a static IP address must be assigned
>> then I recommend using dhcpcd.
>> The "ip" tool is exceptionally difficult to create command lines for.
>> Or at least my impression was that it had a difficult learning curve
>> as compared to legacy methods.
>> I am rather adjusted to it now.
>> 
>> dhcpcd only requires the interface name
>> something like "eth0" or "wlan0"
>> And of course the the network must run a DHCP server.
>> Fortunately, most wireless access points do run a DHCP server.
>> So that should be rather easy to run.
>> Therefore, that makes the most challenging part running iwconfig.
>> Please read the manual page.
>> Using any for the "essid" might work.
>> 
>> # iwlist $INTERFACE_NAME scan
>> 
>> That will list cells that are in range and provide some information.
>> 
>> Learning how to use the tools is a boon.
>> That way a person can take their laptop to an area
>> where there is an unencrypted wireless network.
>> Then they can discover the network and connect to it.
>> Might be as simple as running once followed by iwconfig and dhcpcd.
>> 
>> Once a SA gains the hang of it
>> then it does not seem scary, confusing nor challenging.
>> 
>> My guess is that computers running Microsoft operating systems
>> attempt to automatically configure wireless networking adapters
>> to connect with any unencrypted networking in range.
>> 
>> Sorcerer boxes do no such automatic network connections.
>> It is not a matter of convenience.
>> It is a matter of security.
>> Connecting with a network means that the connection goes both ways.
>> Now other computers have access to the Sorcerer box.
>> Therefore, the SA must be certain that the Sorcerer box is
>> configured properly and securely.
>> The most simple example of that is also the most obvious.
>> The SA must change the root password to something
>> other than merely pressing enter before connecting to a network.
>> Otherwise anyone who can establish a ssh connection or
>> any type of TCP connection to the box might be able to gain root access
>> because the root password is so simple!
>> 
>> So in summary the three command line tools to use
>> in order to set up a trial networking connection are:
>> 
>> iwconfig
>> dhcpcd
>> ip
>> 
>> All 3 commands and /etc/resolv.conf have manual pages
>> that can be read using the man command.
>> 
>> And the templates method for networking exists within /etc/networks.d/
>> 
>> Lastly, Jan Merka knows more about wireless networking than I do
>> and probably so do many other Sorcerer SAs.
>> So be sure to ask them for help also.
