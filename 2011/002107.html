Changes made to /etc/init.d/lilo

As requested /etc/init.d/lilo
no longer removes the modules, initramfs, nor kernel image
for the $( uname -r ) kernel.
Instead the $( uname -r ) kernel becomes the second label.
However, if the $( uname -r ) kernel is also the most recently installed kernel
then it becomes the primary label as would be expected.

Since the most recently installed kernel and the running kernel
are the most logical choices for booting those become the first and
second labels.
If sorcerer-rescue is installed
then that with mandatory password entry becomes the 5th label.
And labels or tables defined in /etc/lilo/tail are added last.

After booting the fast image label may become available.
When that happens then fast is the primary label.
So mostly it works nearly the same but with the modifications
that Jim Decker suggested.
Thanks for the useful feedback.
