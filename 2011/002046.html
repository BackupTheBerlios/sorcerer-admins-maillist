On Tue, Jan 4, 2011 at 11:45 PM,  <Pekka.Panula at sofor.fi> wrote:
> So  after Kyle's messages i am just wondering, are we going towards some
> sort of binary distribution?

I hope not.

> Not that i am against it, because i do use binary distros all the time and
> found them to be working and most of Linux users indeed use binary distros.

Probably the best criticism against deploying Sorcerer
is the amount of time required to deploy Sorcerer
when compared to a pre-compiled binary based distro.

>From my point of view there is no good compromise between
Sorcerer and a pre-compiled binary based distro.
However, some of the deployment time can be reduced by
providing on the Install/Rescue CD
firefox, vlc, and a few other useful programs.
But if Sorcerer SAs seem against that idea
then I will not add those to the I/R.


> Just that is dont take away too much of SA's freedom to make a choices what
> to install.
> I am concerned that our current core software requirements are even now too
> big and we are going towards system where Kyle says what we need.

:)  Actually, it is not me who says what is needed.
The software developers say what is needed.
However, I sometimes have a difficult time deciding
which requirements are optional.

With larger fixed disks these days in most cases
having some optionals as absolute requirements does not
detract from the appeal of a leaner installation.
One might even say that optionals is over-used in Sorcerer
and therefore the configuration of spells before casting
asks SAs too many questions.

> Basicly make more options what to compile and install. Like install just
> bare core for eg. server purposes and if you are workstation user i dont
> mind if theres option for GUI install, etc. But dont require all users to
> install all bloated software, because some of us dont need it and dont use
> it. So there cant be just one size fits all sort of installation because
> IMHO i dont think its possible.

If Sorcerer comes as a full bush
then it can be pruned down to size easily.
But if it comes as a little seed
then it requires a bit of time to grow.
There is a difference between required for initial installation
and becoming permanently required.
The Sorcerer I/R comes with about 300 pre-installed software projects
or maybe closer to 270, and probably from that only about 200 spells
must absolutely remain installed on deployed boxes.

> I am always hated when Sorcerer requires X11 and about "million" spells to
> compile, even though i never need them and even on binary distros there is
> possibility to not to install X Window.

That can be discussed with the gobject-introspection software authors.
It was not my intention to make the installation of xorg libraries mandatory.
It happened to turn out that way.
Therefore, I went ahead and decided that a selection of xorg graphics drivers
and some other xorg related software would make a good addition.
But that was not the first thing I tried.
A long time ago I wrote a patch for gobject-introspection
so that console only boxes could be deployed.
But I did not want to constantly fight gobject-introspection authors about it.

For the most part I do not design the puzzle pieces.
I merely fit them together to create Sorcerer.
So what might look like my decision is usually just
me going with the flow of what the software authors are doing.
And sometimes software is added onto the Install/Rescue CD
because Jan or some other Sorcerer SA tells me that it is deployed
on their production boxes.
If it seems useful for rescue it might just go onto the Install/Rescue CD.

> In fact, mostly our server do run without X libraries and binaries
> installed. Maybe in future when all root access requirements are not needed
> its safe to install them even when you dont need it, well, it's just waste
> of disk space still.

libX11 and a few header files installed is not a huge waste of space.


> Regarding live cd, of course its good to it have a GUI on it but if you are
> installing using it, dont count that we want that GUI to install as a
> mandatory option, leave options to what to install.

A live CD would not be required for an X11 + GUI installation method.
But there were other reasons why I did not want to install that way.
Sorcerer SAs are expected to be able to use a command line
and to use the sorcery, cast, dispel, and augur tools.
An installation that is too fluffy to being with misrepresents
the fact that getting the most out of a Sorcerer box
requires some ability with using a command line shell.

Okay, unless anyone objects
then I will try to make the next set of Install/Rescue CDs more minimalistic
to include prying off xorg libraries.

Looks like the compromise is going towards tiny rather than convenient.
