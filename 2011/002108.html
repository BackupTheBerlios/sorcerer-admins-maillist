gnome 3.0.0 works.
A gnome-session can be with startx when listed in ~/.xinitrc
A gnome-session can be started by running /etc/init.d/gdm
and then logging in.

However, be prepared to wait after entering the password.
There is a lengthy span of wondering.
If gnome-session starts
then something like a panel appears at the top row of a display.
The panel has words and icons that can be clicked
to access menus, log out, lock the screen, etc...
However, beware that locking the screen has semi-predictable results.
First, it might not grant a dialog for unlocking the screen.
It might just blink the screen when the mouse is moved or a key is pressed.
However, if the dialog appears and the correct password is entered
then as expected it does not unlock the screen.
Yes, it is totally broke.
Press ctrl-alt-f1 to switch to virtual console 1, log in, kill
gnome-screensaver,
and press alt-right-arrow until back in X.
Fun, yes?
Until that is fixed
I recommend using something like xlockmore in order to lock the screen.

The most apparent advantages of using gnome would be the larger
default font size
and the huge menu icons.
Makes using gnome from a 1024x768 laptop display easier to see.
However, unlike KDE
gnome's gnome-shell compositing window manager might not
have an option to switch to a non-compositing mode.

A quick summary follows.

Boons: large menu icons; integrated chatting calendar and stuff.

Banes: longer log-in time; broken screen locking program; compositing
always enabled.


Now would not be a bad time for updating deployed boxes
for SAs that are worried about the gnome 3.0.0 update.
However, the screen locking bug might become a painful issue.
