/sbin/bootlogd

It is new to sysvinit.
It provides a nice trick.
I am glad someone coded it.
Perhaps we should use it?

It can be used to create a continuous capture of console output
to a file such as:
/media/root/var/log/console

However while on the subject of console
does anyone still desire the hack that provides
the functionality of redirecting sysvinit and runlevel output to a tty?
The default linux configuration
does not back buffer virtual consoles in RAM.
Consequently, leaving a virtual console and returning to it
eliminates the possibility of pressing shift page up
in order to scroll up to read previously printed text.

Please give objections.
Otherwise I want to axe the old code.
