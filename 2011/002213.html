I want the mode for init-scripts to be less misleading.
Therefore, the mode for an init-script that only
runs during boot/shutdown is 500.
init-scripts that also run during updates is mode 510.
init-scripts that do not run are mode 400.

So now init-scripts will not have the any of the write bit modes enabled.
This makes sense because init-scripts
are not customizable configuration files.
Configuration files for init-scripts belong in /etc/init.d/conf.d/

And previous the group read mode bit was used to indicate
that an init-script should not be run during updates.
Now the absence of the group executable mode bit indicates
that the init-script will not run during updates.

Additionally, I have expanded on the LSB comments.
An init-script can exist in
$GRIMOIRE/$SPELL.d/init.d/ or $GRIMORIE/$SPELL.d/init.d.on/
However, "# Default-Mode: 500" in the init-script
will indicate the init-script should have 500 as the default mode.
Every init-script will have some default mode.
Most will be mode 510.

init-scripts will still retain the mode that the SA sets.
For 6 months sorcery will attempt to mass translate
the mode for existing init-scripts on boxes.
So there might not be anything for a SA to do.
However, if something runs during update that should not
or something fails to run during boot or shutdown
then first check the mode of the init-script.
