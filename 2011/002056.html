The first column is a status column

And the final column displays various information such as duration
or time elapsed since event, or a line from the compile log.

That row shows queue counts.
Cast shows how many spells are casting
Wait shows how many spells are queued to be cast
Download shows how many spells are downloading sources
Acquire shows how many spells are waiting to acquire sources.

[OOo ] is explained in

# augur help scry

It is a visual representation of a bar graph in ASCII
instead of writing a percentage value.
It is not intended to be entirely accurate.
[OO  ] indicates half compilation complete.
However, that gives no clue about whether that is true
or how long the other half will require.
This functionality called "estimation" was requested by SAs.
It might not always be correct.
Most of the time it uses the amount of lines from a previous compile log
as a basis for how many compile lines to expect from the current cast
of that software project.


> What is the *+* sign for?

The + indicates that the time for those is growing larger.
It probably only shows + when software is compiling.

The - indicates that the event happened in the past.
The number follow the - indicates how long ago in the past
the event occurred.
The time is expressed in Days:Hours:Minutes format.
So if there is no : then it is just minutes.

The heads up display shows at the very top something expected for the future.
In the middle it shows what is currently happening.
Then, towards the bottom it shows what previously happened.
When an event has a description such as: Pass Cast
then it is an event that already occurred.
So from top to bottom it is future, present, past.

Sorry if that seemed difficult.
The augur scry screen makes better sense
the more that a SA looks at it.
Because of the limited amount of character space in a console
there is not much point in trying to label the columns.
Most of the columns should be rather intuitive like the spell name column
and the version column.
The first and last columns are more dynamic.
The first column is a status column which might also show status
of an ongoing event as a bar graph.
The last column provides information about the event which might be
lines from the ongoing download log, or a line from the compile log
or a duration for the event or since the event took place.

Many of the augur commands provide verbose help when invoked like:

# augur help $COMMAND_NAME

where $COMMAND_NAME is the name of the command such as:
"event"

# augur help event

Like a HUD on an aircraft airspeed is indicated somewhere
if the pilot knows where to look, however the air number and/or
gauge that shows air speed might not have the word "airspeed" written on it.
So the augur scry, also called the HUD, short for heads up display,
is similar to an aircraft HUD in that it provides information in the spot
where the SA will look once familiar with where to look to find the
desired information.

Unfortunately, it can not show everything.
Compile logs are saved by default.
# augur compile $SPELL
Download logs are not saved by default.
It must be enabled as a feature.
Event logs are stored in /var/log/sorcery/event/
They can be viewed by various methods.
And some of them, which have a size greater than 0
will contain additional information as their file content.
