There exists misinformation about Sorcerer
that implies or states that Sorcerer is on the bleeding edge of
software releases.
However, to the contrary I tend to back off from going with the first releases
of the big version jumps in important software such as gcc, glibc, and such.

The last 3 versions for glibc look like:

stable 2.12.2 2.11.2 2.11.1

Also sometimes I back out from a release when I see it causes a problem.

# augur details telepathy-qt4 | head -n 4
# kdenetwork-v4 version 4.5.4 can not compile with teleapth-qt4 > 0.3.14 ?

    stable 0.3.14
  unstable 0.5.2 0.5.1 0.5.0 0.3.14

I saw the 0.5 series was problematic for kdenetwork-v4 compilation
and so I put telepathy-qt4's stable release versions at the 0.3.x branch
where compilation works.


When I saw the new version of virtual box it looked like x.0.0
Anything x.0.0 predisposes me to expect bugs.
Since many of the Sorcerer boxes run as virtual machines
having Virtual box any flakier than it already has been could become a problem.
I am also predisposed from my experience with qemu to expect
that virtual machine implementations often have buggy releases.
So I advised that waiting for a couple of bug fix releases for 4.0.0
might be prudent instead of rushing on board.
