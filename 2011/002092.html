> Let's just be clear here. I like the automation, and in my humble
> opinion there should be only one script that uses lilo to update the
> bootblock. This way troubleshooting is easy.

Having just one script run lilo would be a good thing.
But there are two conditions for updating lilo
Those conditions are checked for and created by two separate scripts.
Therefore, each of those scripts must be allowed to run lilo.
If /etc/init.d/initramfs regenerates initramfs images
then /sbin/lilo must be run.
If /etc/init.d/lilo regenerates /etc/lilo.conf /etc/lilo/conf /etc/lilo/fast
then /sbin/lilo must be run.

Better to have those scripts run lilo as soon as possible
rather than to wait and leave it to the probability
that a separate script will execute and then run /sbin/lilo

The reason that /etc/{initramfs,lilo} are separate scripts
is so that Sorcerer boxes can be deployed without initramfs
if the SA so chooses to make custom kernels that include
the required hardware and file system drivers.
Therefore, dual configuration requires dual scripts,
and dual invocations of /sbin/lilo
