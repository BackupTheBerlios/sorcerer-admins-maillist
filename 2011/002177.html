I expected that that I already implemented
# augur log event $SPELL
and that it was merely broke.
But somehow I neglected implementing it
or somehow lost the modification.
Yet this is now fixed in the latest sorcery release
along with a fix for # augur delay

# augur event
displays the actual event log file names.
Their file names are very descriptive.

# augur log event $SPELL
displays the content of the most recent
non empty non "yore" event log for the spell.

Therefore, if a spell recently experienced an auto failure
then the string that describes the reason
would be in a "fore" event log for the spell.
In this context "yore" means a historical event
while "fore" means a current event.
Therefore, using
# augur log event gtk+-v3
should explain the cause of gtk+-v3
recent auto failure.
However, if gtk+-v3 most recently cast successfully
then
# augur log event gtk+-v3
would print nothing.
This avoids mistaking a yore event
for an ongoing problem.

Viewing the content of a recent event log
is very much like viewing the content
of a recent compile log or an install log.
Therefore, # augur log event
seemed to be the logical location.
I apologize for not implementing it sooner.
