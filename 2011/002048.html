Evert might have some documentation concerning the menu driven installer.
The items on the main menu are listed top to bottom in basically the correct
order for a SA to select.
A SA without some idea of how to install a distribution probably would not
want to start with Sorcerer.
Obviously, the Sorcerer menu driven installer
explains nothing about what partitioning a device does
nor what LVM does or anything else.
But the SA is free to read the manual pages and documentation
for those that are already installed into the initramfs.

As a compromise to avoid having to read 40 pages of documentation
in order to begin to gain a grasp of the sequences required in order to install
I added to the installer a novice and expert mode.
The novice mode has fewer items on the menu to select.
The novice mode of the menu driven installer
can not do as many types of installations as the expert mode can.
But there is a higher probability that a novice
will be able to accomplish an installation using the novice mode.

The menu driven installer in expert mode probably does not
provide even half the types of installations that are possible
as compared to a skilled SA doing a manual installation from the command line.
But the menu driven installer is a compromise between convenience,
complexity, possibilities.
It is not too easy, nor too complex, nor provide too many possibilities.
It seems to me to be a favorable mix.

I am uncertain about how it is not intuitive.
If I do a manual installation then I tend to use
the same steps and sequence as I would using the menu driven installer.

At one time I explored the possibilities of using an across network mounted
Sorcerer I/R CDROM image, Sorcerer thin client, and sshfs, and httpfs methods
for being able to provide a small Sorcerer I/R while the bulk of the files
were downloaded as needed.
But it is slow.

I also explored the use of a network mounted file system with
a file system union view in order to allow for more software
to appear installed and be usable than what is actually installed.
That is also very slow and very problematic when running ldconfig.

Maybe something halfway usable with an almost decent speed
could be set up using SAMBA running locally to cache requests to
and content downloaded from the server across the Internet.
But I was not pleased with the experiments.

My opinion is that the best experience while installing Sorcerer
comes from having all the required software provided as archives
on the I/R and having the I/R written to USB flash drive instead of CDROM.
