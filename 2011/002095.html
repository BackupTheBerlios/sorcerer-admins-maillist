# echo linux | augur avail
linux   2.6.27.x        2.6.27.58
linux   2.6.30.x        2.6.30.10
linux   2.6.32.x        2.6.32.28
linux   2.6.34.x        2.6.34.8
linux   2.6.35.x        2.6.35.11
linux   2.6.36.x        2.6.36.4
linux   av      2.6.37.3
linux   iv      2.6.37.3
linux   stable  2.6.37.3

This shows that there are several possible version progress to select for linux.
This box has stable version progress selected.
Consequently, the installed, available, and stable versions are concurrent.

For boxes that require drivers from older branches of the kernel
can subscribe to some of those branches that are listed on
http://www.kernel.org/
In order to clearly separate the branch from the version
the branches have names that end with .x

Therefore, reasoning follows that:

# augur progress linux 2.6.36.x
# echo linux | augur avail
Indexing Complete
linux   2.6.27.x        2.6.27.58
linux   2.6.30.x        2.6.30.10
linux   2.6.32.x        2.6.32.28
linux   2.6.34.x        2.6.34.8
linux   2.6.35.x        2.6.35.11
linux   2.6.36.x        2.6.36.4
linux   av      2.6.36.4
linux   iv      2.6.37.3
linux   stable  2.6.37.3

That sets the available version of linux to 2.6.36.4
which is the most recently released kernel for that branch.

# augur progress linux stable

That sets linux's version progress back to stable.

So what happens when a box is subscribed to
2.6.32.x version progress for linux and that version progress goes away?
It might not be desirable,
but it will probably start following "stable."
With linux this is not so much a problem
since 4 older versions are kept in /boot and /lib/modules/
