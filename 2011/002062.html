The word "Fore" in those lines
indicates that the event is considered current and not historic.
Therefore, those sources have yet to be downloaded.
Since many spells appear more than once;
the list is not as long a list as it would seem.

It appears to be just:

eigen
iiimf
iso-codes
kcoloredit
kfax
kmldonkey
lame
libindi0
libssh
lv2core
ortp
rsibreak
sysvinit

Some of those can be forgotten and should not be a problem
because they are probably not being used.
Such as:

# augur forget aron kcoloredit
# augur forget aron kfax

# augur aron iso-codes | grep kde
kde-v4
kdeaccessibility-v4
kdeartwork-v4
kdebase-v4
kdebase-workspace-v4

kdebluetooth4
kdegraphics-v4

kdeplasma-addons-v4
kdesdk-v4
kdetoys-v4
kdeutils-v4

However, a considerable amount of kde requires iso-codes
so it is important that it is installed.
Chances are it probably already is installed.

Fixing download failures first,
Compile failures second,
and that will usually solve the problems.
Also a clue to which spells failed compilation is available by:
# ls /var/tmp/failed
The source trees of spells that failed compilation are stored there.

A considerable number of spells failed to compile including gtk+
which is both odd to fail and important to have installed.
Investigating the gtk+ compile log should be paramount.

A considerable amount of kde requires gtk+
which is probably because

qt-everywhere-opensource

requires it.
and because phonon requires it also.

Almost all advanced X software will pivot on gtk+
so fixing it is required before proceeding to kde-v4.
