> subvol=data,defaults,noatime    0 0

> If we ever start using syslinux for a bootloader, I'll probably use
> btrfs for / as well. With btrfs it's easy to think of your space as
> being infinite, as when you start hitting the limits of your current
> storage, you can just add a disk, or replace a small disk one with a
> bigger one, all while the volume is up and ready for use...
> 
> I feel that this setup allows Sorcerer to grow and it keeps my data in
> one place, easy to back up, and the root Sorcerer install in one
> place, also easy to backup. Root is about 14GB, and can be tarballed
> and compressed down to 4GB with xz. For a complete desktop multimedia
> centre with all the bells, this is not too bad at all!
