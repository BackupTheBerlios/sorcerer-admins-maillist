If cgroup were a sports league
then each Sorcerer box would have 4 competing teams:
daemon; idle; user; sorcery.

And now they have init-scripts:
cgroup_daemon; cgroup_idle; cgroup_user; cgroup_sorcery.
The init-scripts set resource allocation
and provide a menu drive interface for changing
the resource allocation to something other than default.
The current distribution is very balanced
and requires no adjustment.
But it can be easily adjusted if necessary.

cgroup subsystems are mounted underneath /sys/fs/cgroup/
the cgroup file system with blkio subsystem enabled
is mounted as /sys/fs/cgroup/blkio/
Likewise the cpu subset is mounted as /sys/fs/cgroup/cpu/
Using different mounts for different subsystems grants
an advantage of not having a subsystem limitation
apply to all cgroups.
Unfortunately, it is also significantly slow.
