Just before the new month and sometime after my fixes
IANA decided that not only should
http://www.iana.org/assignments/service-names-port-numbers/service-names-port-numbers.xml
but an XML file, but that it should also be served
as a gzip compressed file without changing the file name.
Consequently, the script to download and parse the file
did not anticipate nor gunzip the file as it should.
Some boxes will have /etc/services missing entries until December
unless the SA removes the old file from /var/cache/sources/
and downloads the new one.
I have no clue why IANA is doing this.
If they continue to do so
then I will manually roll a /etc/services file YEARLY.
If I was doing so already
then /etc/services would have remained useful
for 3 months more than it did this year.
