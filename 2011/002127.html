However, "sorcery" makes for a good tool for menu driven
installation and removal of software.

And the "etc" spell provides some useful guidance
for modifying some files located in /etc/

But mostly the root prompt "#" indicates
that the box is ready to perform the wishes of the SA.

Here is the breakdown of the component parts of the command prompt

root at hostname:~#

"root" = the name of the currently logged in account.

"@" = symbol means at

"hostname" = the name of the box

":" = Merely a separator often used to separate hostname from path.

"~" = the tilde is a symbolic representation for the path of the
current account's home directory.

# echo ~
/root

It expands to /root when root is logged in.

When the present working directory changes
then that part of the prompt changes to indicate
the present working directory.


"#" That is the last component piece of the prompt.
When the logged in account is user identification number 0
then the last piece is a "#"
However, when any other account is logged in
then the last piece is a "$"

Generally, "#" is called a root prompt.
"$" is called a user prompt.

In email when I write a command that should be run from a root prompt
then I write the # before the command.

However, noobs that do not know any better
and type the # in front of the command name
will create a comment rather than execute a command.
That much is useful for noobs do not accidentally
wreck their boxes by typing what they see in an email
without knowing what it does.

The most simple of command prompts displays only a "#" or a "$"
Depending upon the shell being used and the content of the variable $PS1
the command prompt may differ.
Also different distributions set the command prompt differently.
On Sorcerer the command prompt requires two lines.
The first line displays the current time and date.
The second line indicates the account name, box, and present working directory.

Search for PROMPTING in the bash manual page
for a complete description of how to create custom command prompts.
