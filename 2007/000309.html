The x86_64 I/R works fine under qemu version 0.8.2
Not certain about vmware.
I have not tried vmware in a long while.

However:

# cast qemu
$ qemu-img create hda 2G
$ qemu-system-x86_64 -m 512 -boot d \
-cdrom /tmp/sorcerer-x86_64-20070905.iso hda

The network drivers are probably ne2k-pci
so modprobe that after it is running
and then run dhcpcd to connect it to your network.
The emulation is very slow therefore it may help to
turn off all services except ntpd.
ntpd you should run, unless you do not mind the clock
inside the simulation running too fast and going ahead
of current time.

Fortunately, with qemu it can hardly be any easier.  :)
