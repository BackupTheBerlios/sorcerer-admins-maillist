On some boxes the IDE fixed disks and CDROMs that normally appear as /dev/hd?
will appear as /dev/sd? and /dev/sg? upon switching from linux to linux-modular.
The above URL provides a decent enough explanation.
However, it does not happen with all IDE devices.
I have a box with a:
IDE interface: Intel Corporation 82801AA IDE Controller
And using linux-modular and it still boots using device named /dev/hd??
If you are using an box with IDE disks and decide to try linux-modular
do not be surprised by what the linux kernel does.
It is a new kernel "feature" and not a Sorcerer related bug.
