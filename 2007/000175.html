Have you seen something like:

13  Compile      poppler      -      57
14  Dispel       ImageMagick  -      28
4   Download     gcc-v3.4     - 1:03:01
5   ELF Failure  kdegraphics  -   01:10
1   Expire       Logs         -   31:34
1   Python-v2.4  -   55:48

And towards the bottom there is that line that has a number
and a spell, but not the name of the log it came from.
Sometimes I see something like that.

Unexpectedly it comes from a bug in find.

Part of the output from the following command:

    find  $EVENT_LOGS           \
          -cnewer $ALL_INDEX    \
          -ctime -1  -type f    \
          -printf "%P$TAB%C@\n"


prints what is below:

Cast Reason/libgnomeui  1187441428
Cast Reason/id3lib      1187441426
Cast Reason/Python-v2.2
Python-v2.4     1187441426
Cast Reason/gconfmm     1187441424
Cast Reason/pcre        1187441429

The line beginning with
"Cast Reason/Python-v2.2"
is missing a change time
The the line below beginning with
"Python-v2.4"
is missing the directory name from where it was found.
