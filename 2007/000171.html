I forgot to mention this from the augur help screen:


merge          spell where Spell always merged from specified grimoire
merge          spell       Merge order for spell not specified

Apparently, in the past I added additional code to allow the SA to specify
the short name ( not absolute path name ) of a grimoire for where a spell
of a specific name should be merged in from.
It could be used to give priority to local spells of the same name
over spells from the official and/or contrib grimoires that may also
be installed.
However, I probably have not tested that functionality since
the day I coded it long long ago and therefore I completely
forgot about it.
You can try it if you want local spells with the same name as spells
provided in other installed grimoires.
