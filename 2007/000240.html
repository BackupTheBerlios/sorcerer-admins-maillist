You may find that some programs behave better
if you do use a locale with a utf8 charset.
However, it does not have to be an en_US.utf8
Once you select an appropriate locale
simply modify /etc/profile
export  LANG="en_US.utf8"
And change that to the desired locale.

# cast -r glibc-locales

And select the locales that you prefer to use.
One of the nice aspects of the utf8 charset is
that you may be able to see properly characters
that you many not normally expect to see given your locale.
I find that locale en_US.utf8 works fine for me to both read
and write Japanese kana and kanji using nano, for example,
although my locale is en_US.
It might also aid in firefox displaying webpages?
And it also probably helps that I have
kochi-substitute fonts installed

In the distant past I had some problem with utilities such as
column not handling multi-byte or high value ASCII characters correctly.
I do not know if the problem still exists.
But maybe column behaves better with the locale set to en_US.utf8 ?
