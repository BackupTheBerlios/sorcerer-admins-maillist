Actually, the current system does support these types of chains.
However, I rarely split spells.

Normally, I would request:

# cast -r -c poppler
And select that you want -enable-poppler-qt-x11-free

However, do not execute the above command.
I will modify the poppler spell to not ask for optional qt-x11-free support.

The next grimoire release will contain spells:
poppler and poppler-qt-x11-free

kdegraphcs will have an optional on poppler-qt-x11-free
That will ensure that poppler compiled with qt-x11-free support
is installed for kdegraphics and not the poppler without qt-x11-free support.

poppler-qt-x11-free will exclusively provide poppler.
Therefore, any spell that requires poppler will be
equally satisfied if poppler-qt-x11-free is installed.

The SA will no longer be prompted if poppler should be
compiled with support for qt-x11-free.
If manual installation of poppler is desired
then the SA should choose between poppler-qt-x11-free and poppler.
However, if kdegraphics is to be installed
and the SA selects poppler-qt-x11-free as an optional requirement
then only poppler-qt-x11-free will be installed and not poppler.
Consequently, kdegraphics will no longer encounter a compile
error when poppler is installed without qt-x11-free support.
