The spell ati-driver-installer is complete.
Additionally, I included the xorg.conf file I am using in
ati-driver-installer.d/
Even if you use the aticonfig program
it will omit including necessary sections.
Therefore, add those missing sections
from the sample I included.

Tricky to install,
worse to configure,
but the DRI performance is adequate.

I tested it on nwn.
With an ATI 9800Pro it does run with a better
frame rate, fewer omitted graphics, and less
slowdowns than it does using the xorg driver.

Chances are I will probably use the driver provided by ATI for a while,
until it melts my graphics card,
or until it the updated version quit working.

As I hoped for it dropped right into place with xorg X11R7.2 installed.
I did not have to compile an old version of XFree86.
I did not have to uninstall any of the spells required by xorg.

The ati-driver-installer spell in the grimoire requires MesaLib which
ensures that MesaLib is cast first
and the post_install() for MesaLib causes ati-driver-installer to recast
if it is already installed.
Therefore, any box with ati-driver-installer installed should have the correct
libGL library installed with appropriate symbolic links.

I am resuming progressing on x86_64 and ia32 I/R updates.
With luck maybe this weekend they will be completed.
Otherwise, look for them the following weekend.

Hope everyone is having a good week thus far.
