Thanks for the report.
I did not expect it was possible that boxes exist where:

/usr/share/i18n/SUPPORTED

was not installed.
Been a while since glibc was updated on that box?

You can either cast glibc to acquire it,
or tear open a tarball of glibc and
# cp  glibc-2.7/localedata/SUPPORTED  /usr/share/i18n/SUPPORTED

Sorry for the inconvenience.
The grimorie no longer contains a substitute for /usr/share/i18n/SUPPORTED

Thanks for reporting the problem.
Since this bug goes away upon
# cast glibc
should it exist on an older deployed box
no fixes in the grimoire will be implemented.
Basically, this problem should occur less frequently
if ever in 2008 through 2038.
It is already expected to be a moot point since
many past versions of glibc spell installed
/usr/share/i18n/SUPPORTED

If the problem persists again after
# cast glibc
Please email me a gzip copy of /usr/share/i18n/SUPPORTED
and what:
# augur version grimoire
outputs.
