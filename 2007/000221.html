On 8/21/07, Pekka.Panula at sofor.fi <Pekka.Panula at sofor.fi> wrote:

> Thats for bugfix. So what will happen now on machines which got installed
> lot of spells under this bug.
> If i now re-cast eg. postfix, does it still override my config if i earlier
> casted it on buggy version because there was no /etc -tracking?

Good question.
The answer is probably not.
Dispel will calculate the md5sum for configuration files.
Then, it will try to find that file and md5sum in all the md5sum logs.
If it does not occur then it assumes that the configuration file has
been customized.
Therefore, it will treat that configuration file as customized.

Due to the spells that have been cast recently that do not have
md5sums in their md5sum logs for the configuration files installed
those configuration files will appear as customized to dispel whether
or not they are indeed customized.
However, upon a second cast of the spell or an update that
installs the exact same configuration file as before
then the new md5sum will match the md5sum for the previously
installed file and the configuration file will no longer be detected as
customized when it really is merely the default configuration file.

More simply put...
You will probably not loose any more customized configuration files
to that bug which is now fixed.
