> First, find out what installed in
> /lib/pkgconfig
> 
> Then,
> # rm  -fr  /lib/pkgconfig
> 
> The directory should not exist.
> 
> About db.
> 
> Perhaps the compile log does not contain the line:
> patching file rep/rep_record.c
> 
> In that case you should probably do:
> # cast -c db
> 
> That way it should compile and the newly compiled db
> should have the correct patches applied.
> Also you may want to disable the reuse feature.
> 
> oracle releases db patches without increasing
> the release version of db, and so do I.
> sorcery itself will detect that the installed version
> requires re-installation, but whether it compiles
> or simple extracts from an archive of a previous
> compile depends on whether the reuse feature is enabled.
> On stand-alone boxes reuse should not be enabled.
> 
> 
> > Then samba does not compile
> 
> No spaces may occur after \
> otherwise \ no longer indicates that
> the line is continued below.
> Whoever added:
> --with-shared-modules=idmap_rid \
> 
> left a few spaces behind the backslash.
