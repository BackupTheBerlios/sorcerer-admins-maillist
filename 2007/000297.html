The box is a laptop with a touchpad which makes the configuration more 
complicated. Your example is similar to what I use for the touchpad but I 
like to use a USB mouse (because of the wheel and extra buttons) so my 
xorg.conf used to have a section

Section "InputDevice"
        Identifier      "USB Mouse"
        Driver          "mouse"
        Option          "SendCoreEvents"
        #Option         "Protocol"      "auto"
        Option          "Protocol"      "IMPS/2"
        #Option         "Device"        "/dev/input/mice"
        Option          "Device"        "/dev/input/mouse3"
        Option          "ZAxisMapping"  "4 5"
        Option          "Buttons"       "6"
EndSection

where /dev/input/mice was enabled in X11R7.2. Now however, it's use results in 
double clicks send to x11. Somewhere I found a mention that /dev/input/mice 
is a hack and should not be used... So a more specific device should be used 
but it changes on re-plug, it's on my to-do list to adjust udev configuration 
for the mouse.
 
With evdev driver you can actually use globbing to identify devices (and do 
more configuration like enabling extra buttons), e.g. here the option "Phys":

Section "InputDevice"
        # Logitech VX Revolution 
        # http://ubuntuforums.org/showthread.php?t=277388
        #http://gentoo-wiki.com/HOWTO_Advanced_Mouse/Individual_Configurations
        Identifier      "Logitech_VX"
        Driver          "evdev"
        Option          "SendCoreEvents"
        Option  "Name"  "Logitech USB Receiver"
        Option  "Phys"  "usb-*/input0"
        Option  "HWHEELRelativeAxisButtons" "7 6"
EndSection
