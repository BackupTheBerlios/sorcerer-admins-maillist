> You may end up with an unusual solution if you do not go entirely
> XFree86 or entirely xorg.
>
> You may first have to remove the appropriate provide logs...
> Perhaps something like:
>
> # grep  -rl  "XFree86"  /etc/sorcery/log/provide | xargs -l64 rm
> # augur index
> # cast xorg
>
> Even then the box may recast many spells,
> some that must be recast,
> others may be selected to recast temporarily while
> required libraries were not installed by XFree86 or
> xorg related spells.
>
> Incase your are curios,
> any spell that that goes with the xorg suite
> will have the attribute X11R7
>
> # augur attribute X11R7 | wc -l
> 296
>
> There are about 300 of them.
>
> If you switch to xorg it uses
> /etc/X11/xorg.conf
> as the configuration file
> which has almost identical syntax
> as XFree86's configuration file.
>
> If you are looking at the old one
> running xorgconfig will probably
> help you create a new one that xorg can use.
>
> You can use XFree86, older version of XFree86 or xorg.
> However, the test box tests and adjusts software to compile with xorg.
> There is the possibility that by using XFree86 or older versions
> that you may encounter problems compiling some software.
> A version or two ago of qt-x11-free would not compile with the
> current version of XFree86.
> Whether that has been fixed I do not know.
