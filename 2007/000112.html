spells linked against mesa will not recompile when Nvidia is casted 
because both provide opengl api and link against libGL.so and libglx.so 
who are standardised by opengl.org... both spell provide that lib and no 
version information is stored in the elf header which means that simply 
modifying the symlink make you use one or the other lib.... if a user 
accidentally select NVIDIA-Linux-x86 then it should compile fine but glx 
won't load on X11 start.... the user can then select the other 
implementation without any trouble
> There may be, in the grimoire, a spell for another somewhat more stripped
