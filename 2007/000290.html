100.14.19 is the version in the grimoire
and the version offered by NVIDIA.

Adjusting the install location of the NVIDA files may cause additional
malfunctions
when trying to run the server.

In the files section of

/etc/X11/xorg.conf

add the following line

ModulePath "/usr/lib/modules"


Then, the server should be able to locate the NVIDIA-Linux-x86
installed modules.


I have not tested NVIDIA-Linux-x86 version 100.14.19
with xorg X11R7.3
Since I do not have NVIDIA hardware I would not be able
to start an X server with those drivers anyway.
However, until someone tells me they do not work
I will assume that X11R7.3 and NVIDIA-Linux-x86 are compatible.

Similar to the NVIDIA-Linux-x86 hang problem
the i810 drivers provided by
xf86-video-intel version 2.1.1
with xorg X11R7.3
appear to lock the X server in an unusable state
upon logout to kdm.
Also switching to virtual console will probably lock it.
Trying to zap the server probably locks it.

So far I have yet to have anyone report that the
upgrade to X11R7.3 has provided improved stability
or improved performance over X11R7.2
I also am somewhat disappointed.

I am strongly considering making an optional switch for xorg
to enable all the xorg related spells to be rolled back to
X11R7.2 for easy regression.
Some of you may already be trying on a more spell by spell
basis to restore X11R7.2, which would be difficult and time consuming.
Using X11R7.3 seems intolerable on boxes where the
X server locks up during logouts.

I am not trying to make administration difficult for SAs.
I leave that task to software developers.
With sorcery I try to make deployment and upgrading of
installed software on boxes as easy as possible.

Obviously, xorg developers slacked off a bit with the X11R7.3 release.
But it might still be useable on some boxes.
For now I will try to implement a compromise so that those of us
pained by X11R7.3 do not have to continue to endure it.
