It is not a problem that will go away with recompilation.
I have seen it for some time now, maybe longer than a year.
It might have something to do with svn
Even if svn is terminated with signal 15
that probably should not create an adverse effect.

# svn commit -m "comments from a bug report"; echo $?
Sending        gdm
Transmitting file data .
Committed revision 9149.
Killed by signal 15.
0

15 is a normal termination signal rather than an error
such as being killed by SIGHUP, SIGINT, SIGABRT, SIGSEGV.

As I demonstrated in the above example
yes it prints Killed by signal 15.
However, it does not return non zero.

# false; echo $?
1

If it returned 1, such as above
then that might cause problems for something
that checks it's return code.
