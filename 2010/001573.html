Yeah, I see the need to have a boot disk that boots everything, and also the
need not to have all that in the kernel.

However, I run the monolithic kernel because I can actually turn off
exisiting hardware that I do not want to use, and end up with an incredibly
lean-fast booting kernel. All of it fits inside my L2 cahce on my processor,
so it's lighting fast.

It's still possible to go from modular to monolithic, but it's a skillcheck
for an SA. Maybe one day I will write a howto... Once I decide to install
off an I/R again.
