xz is an implementation of LZMA using a libz type frontend.
Ya know, like gzip, but higher compression than bzip2.

Guess I created a chicken/egg situation?
The I/R is too old.
I only upload xz compressed sorcery tarballs.
Naturally, they require xz be installed software
otherwise they do not decompress.
Sorry.

I want to create new I/Rs soon.
For now download xz,
decompress, untar, compile,
and temporarily install into /usr/local/

http://tukaani.org/xz/xz-4.999.9beta.tar.bz2

Sorry for the inconvenience.

Oh about dhclient..
Try:
# dhcpcd eth0
or
# dhcpcd wlan0

xz is the preferred archiver now.
Smaller sliding windows can be used
in order to achieve faster yet less efficient compression.
Or larger sliding windows can be used to create better
compression than is capable with bzip2.
Also xz decompresses faster than bzip2
and sometimes faster than gzip.
