Again using KDE screen locking is unsafe.
I was able to repeat this error three times in a row.
I locked the screen.
Then, I pressed, enter for the password.
The first time, it printed failed as it should.
Then, I pressed enter again for the password.
And I arrived back at the desktop as if the screen had unlocked.

drkonqi[4373]: segfault at 0 ip b7844a90 sp bfc308e0 error 4 in
libkdecore.so.5.4.0[b77d2000+245000]
drkonqi[4381]: segfault at 0 ip b7808a90 sp bf9d5500 error 4 in
libkdecore.so.5.4.0[b7796000+245000]

Lines such as those were in the system log and kernel ring buffer.
Please either log out or use an alternate x locking program.
If this bug is not repeatable in your KDE installation
then please tell me what version of KDE is installed and running.

For those who want to use xlockmore temporarily
increase the stable version to 5.30 and it will be able
to download and cast.
Currently, the directory structure used to store releases of xlock
complicates the automatic discovery of new releases
since new releases always appear in a separate directory
with a name not known in advance.

I apologize for the inconvenience.
This appears to be a bug in the KDE 4.4.0 release
that the KDE authors have provided as a stable release of KDE.
I do not know if the bug exists in the 4.3.x KDE releases.
Please be certain to check rather than assume it is a version 4.4.0 only bug.

I take screen locking for granted.
I do not waste CPU cycles on screensaver programs,
no matter how eloquent and mesmerizing.
However, my favorite type of lock is not slock, but glock.
Light, fast to field strip, easy to clean,
trigger pull as light as a single action,
but has the convenience of double action re-cocking.
What's not to like?

Speaking of not liking....
Looks like KDE stopped releasing security patches
sometime during the 3.5.x releases.
So for SAs with lots of KDE deployed issuing a department wide
email to use a different screen locking program
is a huge inconvenience and potential security problem.
I can understand wanting to blame it on the Sorcerer team,
but KDE is not software that Sorcerer authors wrote.
We just make it compile and install.
And the quality after that
is the responsibility of the software authors.

Seconding guessing the KDE team is not something we should do,
because seriously, they know the guts of KDE
while we are just KDE users.
So the 4.4.0 release was going go distribution wide
merely because the KDE software authors stated it was stable.
As soon as KDE authors make a new release
I will try to make it available and hopefully
that will fix the crashing of the KDE screen locking
and error handling programs.

Oh, there it is.  KDE 4.4.1
Woah, faster than pizza delivery in the 1980s.
I will be testing that today and tomorrow.
Chances are the requirements are nearly the same.
So if you want to try it sooner than later feel free
to give it a compile.
I hope screen locking and konsole crashing bugs are fixed.
