Games work.
Even games in wine work.
With Sorcerer IA32 installed in /opt/ia32/
then wine runs fine chrooting, provided:
the directories to ELF32 libraries are in /etc/ld.so.conf;
/lib/ld-linux.so.2 is a symbolic link to /opt/ia32/lib/ld-linux.so.2;
and PATH includes /opt/ia32/usr/bin.
Why might someone want to install and boot Sorcerer IA32 exclusively
if the box can run Sorcerer x86_64?

However, because linux began on 80386 for 32-bit machines,
because there is some pre-compiled software for linux 80386,
and because there is plenty of pre-compiled win32 software to run in wine
then the continuation of Sorcerer IA32 seems a necessary compliment to
Sorcerer x86_64.

However, the zero-install injector team might also have a solution.
I have not tried the software, but have long been fond of the concept.
They provide a method for installing from packages pre-compiled software
within a user's home directory for nearly immediate use.
It is similar to what I originally wanted for sorcery,
which was a tool for source based administration
on an installation of Debian.
That was back before I rolled the first bootable Sorcerer Install/Rescue disks.

Alternately, those who want to run a few win32 apps
could use a virtual machine emulator
for installing a Microsoft operating system.
Of course, that is a Faustian deal,
but then again most people pay the Microsoft tax
when purchasing a new laptop
even when they decline the EULA
and wipe the fixed disk clean.
