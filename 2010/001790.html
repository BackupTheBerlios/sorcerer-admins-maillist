gobject-introspection is a bit of a pain.
The patch was to attempt to make gobject-introspection
to be able to compile without cairo support
and without libX11 support.
If cairo is required
then a large portion of xorg is installed with it.
Bit of a bummer.

The patches used to work.
I guess a more recent version rendered them useless.
I made the patches for version gobject-introspection
version 0.6.10 and forgot to upgrade/test them
for version 0.6.14
This is a fantastic reason why
Sorcerer maintainers should NOT fix/hack other
software authors sources.
I break my own guideline and it bites me in the arse.
Install cairo and libX11 prior to casting gobject-introspection
and the problem will go away.
Sorry for the inconvenience.

Ah, gobject-introspection is a new gnome type thing
and it was not introduced properly and so there are
some sources like gobject-introspection that
try do more than merely install the utility,
but it also creates gir files for other sources
when those other sources should be invoking
g-ir-generate to create their own gir files if gir is installed.

Trying to create a console only box is too difficult
since a little gnome foot is always blocking the door.
