Ah...

Yes, that might do it.

Ok..

Go to:

http://sorcerer.silverice.org/download/grimoire/

Get the latest grimoire, and save it in the proper subdirectory in
/var/cache/sources

for instance... latest one I have is:
grimoire-2010-92.tar.xz
in the directory
/var/cache/sources/grimoire/2010-92


I would highly recommend downloading the bz2 tarball, instead of the .xz
tarball until.


Now, issue

cast grimoire


This should give you the new grimoire installed.
Next:

cast sorcery


then you should be able to issue

augur sync


after which

augur newer

should work...


after which

augur update

will get your system up to date.

-----------------------

Hope some of this helps?
-Evert-

> Hi there.
>
> I thought you had Sorcerer installed some time ago.
>
>
> Do you mean you got X11 and KDE installed?
>
> The sources not downloading, while highly unusual, does sometimes happen.
> You got the sources for everything else so far, so I assume that your
> network connection works properly.
>
> Every time I had this same thing, I was aborting spells while they were
> casting, etc.
>
> Just about the quickest way to see what is happening, is to reboot.
>
> Once you are back up, type:
> *augur clear*
>
> Then, type :
> *cast firefox*
>
> When it shows you that it is waiting to cast firefox, press *q*
> You will now be back at the command line. Issue:
>
> *augur scry*
>
> Now you will see what Sorcerer is doing.
> I highly suspect that there is a dependency of firefox that is not casting
> properly, making firefox autofail.
>
> If the same thing happens again, type
>
> *augur activity failed*
>
> Go to the bottom of the list... It should say firefox autofailed because of
> blablabla
>
> Once we isolate what is not working on your box, we can get to get that
> source compiling.
>
>
> Check to see if the firefox sources are available:
> *cd  /var/cache/sources/firefox*
> Check to see if there is a directory under there with the latest available
> version number of firefox, and that that directory contains a tarball.
>
> *root at dora:~# augur available firefox
> Indexing Complete
> 3.6.2
>
> Sat Apr 03 03:54:25
> root at dora:~# cd /var/cache/sources/firefox/3.6.2
>
> Sat Apr 03 03:54:42
> root at dora:/var/cache/sources/firefox/3.6.2# ls
> firefox-3.6.2.source.tar.xz
>
> Sat Apr 03 03:54:44
> root at dora:/var/cache/sources/firefox/3.6.2#
> *
> Lastly, remove any half-leeched sources from /var/tmp/leech
>
> rm -r /var/tmp/leech
>
> Then try again to get firefox.
>
> cast firefox.
>
>
>
> In /root/.sorcery/grimoire should be all the spells that are currently
> available in your system.
> If you were to inspect the firefox spell, you can see where it's trying to
> download the sources from, and as a last resort try to download the sources
> yourself, and place them in the source cache.
>
> Lastly, I don't believe Sorcerer has a gui.
> I think what you are referring to as a gui is "augur scry" in a terminal
> with a transparent background. I also like to run gkrellm, as I like to see
> what my computer is up to while compiling.
>
> Hope some of this helps!
>
