There is some confusion about linux*

What was linux is now linux-custom.
What was linux-modular is now linux.
And of course there are several linux-2.6.* spells also.
Why?
See what is listed as stable at http://www.kernel.org/
That is why.
The linux spell will provide a modular kernel for the latest available
linux kernel.
The older kernels that are still available, listed as stable and maintained
are available with modular config for the other linux-2.6.* spells.
The reason for the older kernels is because some hardware drivers
break and only work in the older kernels.
Who knows when the fixes will move forward?

The new I/Rs will come with the linux kernel pre-compiled.
It has all the drivers, experimental and stable, for fancy new hardware.
The linux kernel to install on boxes is whichever kernel the SA wants to cast.
Once a linux kernel is installed
then it will take over as the linux-kernel provider.

A few boxes might have both gcc and gcc-core installed.
Eventually those boxes should only have gcc-core installed.
They should fix themselves.
But if the box already has gcc-core installed
and the SA wants to expedite the fix
then dispelling gcc is safe after gcc-core is installed.
gcc-g++ is also good to have installed
to provide for libstdc++ for c++ sources.

Because some SAs prefer to start from a cleaner loadout
