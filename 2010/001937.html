If pvcreate was previously executed on /dev/sda1
then the execution of /sbin/vgchange --ignorelockingfailure -ay
would make /dev/sda1 unable to check and unable to mount
if the expected bug in LVM2 exists.
It might not be worth checking on that laptop,
because if the bug exists then
/boot would no longer be normally mountable.
However, the problem could be fixed after booting
by blanking out the start of /dev/sda1
and creating a new file system for /boot
and casting lilo and linux.
