I also agree that init-scripts that require variables
should source those in from a separate modifiable configuration file.
However, in the past there has been a lack of consensus about that.
A few years ago SAs wanted everything in one file.
Therefore, with maybe one or few exceptions each init-script
has any or all of it's variables defined within the init-script.

There are also init-scripts such as /etc/init.d/kbd
that is modified by the spell before it is installed.
That way the SA can enter the keymap to use
during the configuration portion of the spell
and have that automatically go into /etc/init.d/kbd

If many of the init-scripts do not work as is
then they should be revised to be better.
I would not be opposed to seeing
a separate directory such as
/etc/init.d.conf/
that contains files with the same name as an init-script
yet have those files are automatically sourced in as a configuration
file for the init-script.
Therefore, files in /etc/init.d.conf/
would be customizable configuration files.

Assuming we go with an implementation such as above
then what init-scripts require improvement please?
/etc/init.d/httpd we can definitely do.
Any more specific information about
what variable or variables should be used where/when, please?
