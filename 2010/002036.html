No, not exactly.
Removing /lib/libc.so.6 and the symbolic link's target file disables multilib.
The rest of the software loses it's ELF32 libraries when it next casts.
I use
[[ $HOSTTYPE == x86_64 ]] && [[ -f /lib/libc.so.6 ]]
as a very simple check to see if a box is multilib.
