Hi,
on my box gnome-doc-utils install it's pkg-config files into
/usr/share/pkg-config (there are several .pc in this directory) thus when
gdm try to compile, pkg-config don't find the .pc file and stop the
configure....
I solved this issue by moving the .pc into /usr/lib/pkg-config dir and doing
a symlink to re-create the /usr/share/pkg-config dir.....
I've no idea what caused the installation to not install the files into the
usual pkg-config dir... however I think that /usr/share/pkg-config shall be
a symlink to /usr/lib/pkg-config and I propose the pkg-config spell to
create it at install step
