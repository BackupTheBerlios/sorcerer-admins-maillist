On Wed, Feb 3, 2010 at 9:23 AM, nathan_allen1 at juno.com
<nathan_allen1 at juno.com> wrote:
> could someone tell me how to create a new sorcerer partition?

First, a disk block device must have some space for a new partition.
Having partitions of size 0 is usually useless.

Then, one should run fdisk on the disk device.
If the disk is the first IDE device then fdisk /dev/hda
If the disk is the first SCSI device then fdisk /dev/sda
Just check /dev/ for devices.
udev will automatically create nodes for both block devices
and partitions as they are discovered.

Now, for an empty disk I would normally type something like:
fdisk /dev/sda1
n
p
1
<enter>
<enter>
a
1
w

That would be new, primary partition, partition 1, start 1, maximum
size, a to toggle it to bootable, 1 to specify which partition, and w
to write the changes to disk.

Typing ? in fdisk shows the help.

A partition is merely a way to divide the content of a fixed disk.
There are other methods also.
Once a disk has a partition
then a filesystem can be created on the partition.
