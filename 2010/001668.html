With XFree86 and earlier versions of xorg
there was a program that could create a configuration file
by having the SA answer a series of questions
which would include information such as graphic
chipset and monitor horizontal and vertical frequency ranges.
But perhaps that program is gone now.
I can not seem to find it
or forgot what provides it.

if
# Xorg -configure
hangs
then I would probably try
copying and editing an older configuration file
or configuration file from a different box.
Or reading the xorg.conf manual page
and manually creating an xorg.conf
Or I might try installing an older or newer
version of xorg-server.

Also the pointer input device can be re-selected
one Xorg is running.
Hmm, well xinput used to be able to do that,
but the manual page now says:

"       --set-pointer device
               Switch  device  in core pointer.  This option does nothing on X
               servers 1.5 and later.
"

Xorg has a possible useful parameter:
"-pointer name          specify the core pointer InputDevice name"

Maybe there are other useful Xorg parameters.

Is Xorg -configure hanging
or is perhaps it completing and exiting,
yet the driver is failing to properly
restore the video mode for consoles to work?
If so the box might still be accessible via ssh.
That might be the case with chipsets that
use Xorg's "intel" driver.

Want for me to try to generate an xorg.conf,
gzip, and email it to you?
Then you can start Xorg without the configuration file,
see what drivers it is using,
fix the spots in the xorg.conf
and then you will have a usable one.

The version 1.8.x and later xorg-conf
support an /etc/X11/xorg.conf.d/
directory where fragments of configuration files
can be inserted.
No I have not tested it.
But perhaps with later xorg-server
one can provide a fragment of an xorg-conf
file that only configures the core pointer?

Later versions, such as 1.8.1, of xorg
are available in the latest version progress.
They are not unstable nor stable.
Who knows when the next full stable release
of xorg will happen.
When it does then all of the stable version progress
versions will be updated as required for spells
with the X11R7 attribute.

Good luck.
