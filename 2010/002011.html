Hi

Good question, because almost all other major distros do not require /boot 
to be so big. 
I think because Sorcerer's way to make initrd is to copy all stuff there. 
I think it even might copy all kernel modules too, even though you should 
only need those that your storage can be found. Not sure about that but 
because its so large i assume its basicly own distro what its creating 
there per every kernel version. 

Only purpose to whole initrd is created that Linux finds where your 
storage and your installation is.
You dont actually dont need any initrd if you compile all needed drivers 
direct to kernel, thats the best way, fastest too.

But because most people wont compile their own kernel (but Sorcerer do, so 
whats the point to use initrd system as default?), its needed some modular 
way to load block drivers and sometimes network drivers if you use 
something like NFS boot or iSCSI boot that boot process and kernel finds 
your storage. 

I surely hope that some day Sorcerer's boot process will be much lighter 
than its on currently. Kyle might say that there is no slack space but 
IMHO it can be designed to be smaller.

I have several older installation where /boot is smaller, so in worst case 
situation i have here that not even two kernels can be installed there...
