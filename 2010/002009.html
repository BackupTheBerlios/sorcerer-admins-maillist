I am considering buying a motherboard with integrated ATI graphics tomorrow.
So I decided to pound on the ati-driver-installer spell today.
Unlike previous versions this one did not segfault while running the
installer or extractor.
However, it is not fully compatible with linux version 2.6.36
I found some patches, sedited some files, adjusted the spell for DESTDIR
and after all that it seems to be installed.
If anyone wants
then please feel free to give it a try.
The ati-driver-installer spell that went out in today's LAG=0 grimoire
is incomplete.
It will not track the installation of the module file in /lib/modules/
However, tomorrow's ati-driver-installer will.
Either one should be usable to start an X server if it will work.
If not I guess it is SVGA instead?

I know that every few years I skip over from using ATI to NVIDIA
then after a few years skip back.
I considered buying an Intel core i3,
but then again I already know that xf86-video-intel is no picnic.
At least for the moment NVIDIA-Linux-x86 seems good to go.
I am posting this information in case anyone has an ATI chipset
and wants to try it with the ATI/AMD provided driver.

Also please be aware of the rovclock program.
It can make possible using some ATI cards
that would melt the GPU fan or overheat.
The rovclock can be used to lower the clock frequency of the GPU.
And while nobody ever talks about under-clocking
the performance hit is not at all a bad compromise for reducing heat production.
