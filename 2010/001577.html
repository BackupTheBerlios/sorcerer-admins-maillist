I am trying version 195.36.24
to see if it still does like version 195.36.15.
Unfortunately it does.
The other day I was notice a high swap usage,
yet it should not have been.
However, X was using 1.5G of swap.
My guess is the ever increasing memory consumption
is due to the NVIDIA drivers.
So should one notice that X is using more than 100M of swap
then restarting X might be a good idea to prevent unnecessary
wear and tear on the disks.
Also the usage can probably go into the gigabytes
until eventually the X server crashes.
I might be wrong about the problem being caused by NVIDIA.
So if anyone notices it happening on non NVIDIA workstations
then please report it.
It might be solvable by careful upgrading of xorg software versions.
