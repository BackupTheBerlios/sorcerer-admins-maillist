After today's daily grimoire is rolled
which happens at UTC 00:00
I am going to update the I/R images and rush
them to sorcerer.silverice.org for download
in order to repair the problem with /etc/init.d/find_ir
in the event that a floppy disk device is detected during boot.

This bug might be possible for seeing later bugs appear
during the menu driven installer such as the programs such as
sort and uniq missing.
It could explain unexpected weirdness with the menu driven installer.

Here is the immediate solution to the problem...

Start booting the I/R
When isolinux asks for the command line type:

linux rdinit=/bin/bash

It will boot and immediate present a prompt
now type:

nano /etc/init.d/find_ir

   try(){ while read  &&  mtest  "$REPLY" && return; do :; done; false; }

That function is incorrect
Replace that entire line with

   try(){ while read; do mtest   "$REPLY" && return; done; false; }

Now press ctrl-x y enter
and the corrected file will be saved.

Now enter the command at the prompt
exec /init

The I/R will now continue booting normal.
Login as root with no password
And start sinstall

Hopefully, with this correction
installation will work as smoothly as it does for me in qemu.  :)

If this fix also corrects any weirdness encountered when running sinstall
then we can safely assume that the bug that required fixing was not in
the menu driven installer, but instead within the /etc/init.d/find_ir script.

I will post another message to the list
when the updated I/Rs are ready for download.
