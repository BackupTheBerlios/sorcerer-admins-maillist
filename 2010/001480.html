Apparently, running pulseaudio during runlevel start
would not be a good idea for several reasons.

http://pulseaudio.org/wiki/PerfectSetup

That might be useful, but it is also a considerable read.

Here is the part about kde

"KDE 4 uses Phonon as the main audio interface. The Xine backend of
Phonon should eventually use PulseAudio automatically, but at the time
of writing the pulse plugin for Xine is too unreliable, so it's
disabled by default. While waiting for that to get better, Phonon uses
Alsa. Therefore, to get Phonon to use PulseAudio, you have to edit
your ~/.asoundrc or /etc/asound.conf. However, the normal .asoundrc
modifications aren't enough. See #232 (specifically the workaround
part). "

There is also a section on using it with xine.

The docs seem to suggest that unless connecting to a system
wide instance of pulseaudio that membership to a pulseaudio
or pulse group is not required.
pulseaudio can and should be started as required per user.
With kde, it aught to start automatically.
