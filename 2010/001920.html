That would probably be the docs feature toggle.
sorcery -> Config -> Toggle Features

It does not have a smart yet CPU intensive way to
decide if docs were installed elsewhere.
So according to some rules
it gathers up anything that might be a document
and places them in /usr/doc/$SPELL/
This feature was requested by SAs
that noticed that the make install
of several sources would not include documentation.
So this feature provides a convenience,
but also wastes disk space.

The most recent revision to that feature is the feature
to install documentation into /usr/doc/$SPELL.tar.bz2
or something like that.
Saves a little bit of space while preserving part of the convenience.

However, when I want to read some docs that
are included in a source tarball but not included in the install
then I usually untar the source tarball and start grepping.
I expect that this is best since
Sorcerer boxes normally retain
source tarballs for installed spells
in /var/cache/source/
/var/cache/source is cleaned of debris regularly
by the sorcery-clean spell.
Costs about 2 to 3G of worthy amount of space on an average workstation
when sorcery-bxz is installed.

Please be careful of which features are set.
They often make changes to how sorcery works
that are much less than subtle.
Some like the "retry" feature can even become problematic.

The default feature set is color and scavenge.
sorcery will work fine with no features enabled,
but it will look less colorful.
