My best guess is that Jan already has a solution to this.
He reported some sound problems with kde-v4 4.3.5
If I recall correctly he said something about installing
an old phonon over qt-x11-opensource's phonon was
sufficient for solving that problem.
However, now qt-everywhere-opensource does not install phonon.
And the phonon installed is version 4.3.80 from KDE's svn.
Sound seems to work for me,
no crashing of xine.
But if xine was crashing then that might explain
a problem with audio.

# augur elfof phonon | grep xine
xine-lib

phonon uses xine as it's backend for audio output.
Perhaps it might also be able to use pulseaudio.
I do not have pulseaudio installed.
And that might be why audio works on my workstation.

/etc/xdg/autostart/pulseaudio-kde.desktop
/etc/xdg/autostart/pulseaudio.desktop

Perhaps pulse audio server is automatically started

# grep user /etc/dbus-1/system.d/pulseaudio-system.conf
  <!-- System-wide PulseAudio runs as 'pulse' user. This fragment is
       not necessary for user PulseAudio instances. -->
  <policy user="pulse">

But maybe it requires a special user account for it to work?

If pulse audio is installed
then I would recommend
# augur forget pusleaudio
# dispel pulseaudio
# cast eject

Following the cast of eject cast will
re-compile the spells which require pulseaudio's ELF.
After that I would recommend logging out,
restarting kdm,
logging back in
and then trying audio.
If it works, maybe pulseaudio is the problem.

I will have a glance at pulse audio documentation
to see if I am omitting something important that
the spell aught to do to ensure that pulseaudio can work.
Generally, when possible I try to avoid software
audio channel mixers such as
enlightenment sound daemon and pulseaudio.

However, foremost I recommend investigating
and solving the xine segfaulting on the box.
Fixing that might be enough to fix the problem.
