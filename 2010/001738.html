Oops, sorry about that.
Spells are case sensitive.

# cast xorg gnome

Why not configure them both and put them
into the queue so that you can go out for lunch
while it compiles.
It is a very heavy compile and likely
not all of gnome will compile,
but enough of it should install to be usable.

If then you want a GDM login prompt on boot
edit /etc/inittab and change the default runlevel
to 4 or 5, depending upon your preference.
A description of the runlevels is in /etc/inittab
Not certain if that description is correct.
However, runlevels 0 1 3 5 6
are the same as they would be in any distro.
2 is like 3, but with parallel execution of init-scripts.
4 is like 5, but with parallel execution of init-scripts.
Provided that the disk drive can keep up,
and it will probably bottleneck booting,
using runlevels with parallel init-script execution
can be a bit faster to boot than sequential.
But it might only shave off a few seconds.

If a problem that the SA wants to debug occurs
during runlevel execution
then such problems should be debug using
a sequential init-script runlevel.
That is the only way to be certain that an
error happened while a certain init-script was running.
The logs runlevel log for 4 and 5 look different,
merely because of when init-scripts start and complete.
