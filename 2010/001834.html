Hmm, someone must have not liked how
it overwrite /etc/modules?
Best guess is that someone removed
the file from module-init-tools.d/
and so now that file might not be tracked
on some installations of module-init-tools.
Therefore, next time module-init-tools casts
it will overwrite /etc/modules on every box.
And ones that did not have a tracked /etc/modules
will not have a preserved /etc/modules
Good thinking.  :)
I fixed the spell.
I added module-init-tools.d/modules
Everyone should make a copy of their /etc/modules
if they definitely want to keep it.

If spells have config files
then those config files should be installed
and should be tracked.
Tracked files receive a md5sum.
If the config file is modified
the md5sum will be used to detect that
and preserve the modification.
Files not being tracked receive no md5sum
are not preserved when modified
and are overwritten.

Thanks for the report.
