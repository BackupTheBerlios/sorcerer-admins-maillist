The sorcery spell that went out with grimoire version 2010-182
is not able to install the newer sorcery.
Sorry about that.
Well, it depends.
Boxes with sufficiently older sorcery will install it.
Boxes with newer sorcery that have only partial support for DESTDIR
will install it into the wrong place and it never reaches /usr/libexec/sorcery/

So I increased the available version for sorcery to
kame-20100630-1

And I added a more complicated build() that checks for
an incorrect DESTDIR and exports .... well it looks like:

build(){
 if [      "$DESTDIR" == "$META_DIR/$SPELL.install" ]
 then export DESTDIR=/
 fi

 DESTDIR=$DESTDIR ./install $VERSION
}

Sorry for making a pain out of the transition.
