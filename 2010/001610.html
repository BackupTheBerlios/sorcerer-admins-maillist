On Thu, May 13, 2010 at 1:37 AM,  <Pekka.Panula at sofor.fi> wrote:
> Updating older installation and i am now getting this:
> Warning: The initial RAM disk is too big to fit between the kernel and
>    the 15M-16M memory hole.

How old can the installation be?
The default lilo.conf.head that comes
with Sorcerer I/Rs contains the
large-memory line
Add that to /etc/lilo.conf.head
and the warning message will go away.

And yes, large initramfs images are loadable
on architectures as old as Pentium MMX 233Mhz
The BIOS would have to be very old broke
for at problem to occur.

> Is it safe to boot?

Probably.
At least what you showed was only an warning message.
I still recommend editing /etc/lilo.conf.head
and then running /etc/init.d/lilo.conf

> Also for some reason sorcery always wants to "update" my gcc-core, its old
> machine so its sort of PITA when its always wants to casts it...

PITA?
So other software must be updated first?
gcc-core should only require casting once.
Could some alien files be causing a problem?

> It also has couple <spell>-failed "installed", how should i get rid of them?

Remove them manually.
# rm /etc/sorcery/log/*/*-failed
Then:
# cast sorcery-alien
And then look at the compile log for sorcery-alien
and move all the aliens to somewhere safe
where they can cause no strange behavior.

> dispel certainly does not remove them, they always come back.

Dispel probably can not remove the
*-failed spells because the *-failed spells
install logs are incorrect.

Definitely, the box should be updated
and have the aliens weeds pulled out.
After that it aught to be okay.

While you are at it, please check to be certain
that /etc/lilo.conf.head exists
And if it does not then copy one from a newer
installation and edit it.
I wonder how many problems might occur
if the ELF logs are 1st or 2nd generation
or entirely missing?
