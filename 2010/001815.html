> One can always press q
> to quit the complimentary augur scry
> if one does not want to wait and watch.
> In order to see what is actually happening
> in another terminal emulator or virtual console
> while logged in as root or su - or sudo su -
> then type
> # augur scry
> 
> When augur scry is invoked without a spell name
> then it shows a heads-up-display of ongoing activity.
> To check if cast has stalled try:
> # pgrep cast
> or
> # ps -Af | grep cast
> 
> If everything is waiting and nothing is happening
> then that might indicate that cast has stalled
> or that the box is waiting on leech to download
> some source files before starting cast again.
> When many spells are queued for casting
> then cast will run if it has something to do.
> However, if a box is waiting on sources
> then the last instance of cast will exit.
> Then cast will be started again by leech
> when a source is successfully downloaded.
> 
> Some SAs like to watch compile scroll.
> As for me, not so much.
> However, typically when
> # cast $SPELL
> is given then it will switch to
> # augur scry $SPELL
> After cast has finished asking the SA
> questions related to configuration and optionals
> then cast goes background.
> # augur scry $SPELL
> is foreground.
> Pressing q merely quits the view.
> It does not effect compilation.
> 
> Please remember to not use ctrl-c
> to try to exit from any of the sorcery tools.
> The tools do not like that because the ctrl-c
> could interrupt at a very inconvenient time
> and spoil the merged grimoire and generated index.
