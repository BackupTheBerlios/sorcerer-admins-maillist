The new ccache is sweet for expediting recompilation.
It is a bit faster on the first pass.
And it is between 3 and 5 times faster on the recompile.
That is because ccache concatenates files to make a hash
instead of using gcc's preprocessor to do so.
Also the ccache provides on Sorcerer
is statically linked for extra fast startup.
If a box is low on CPU or
tends to heat up and throttle down
Then ccache might be able to help in the long run.
