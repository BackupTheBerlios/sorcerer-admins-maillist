The linux-modular kernel configurations already have:

CONFIG_X86_PAE=y

So for the reason of wanting to use more than 4G
a custom kernel configuration is not required.

Likewise the kernel configuration for x86_64 kernels
provided by linux-modular also support 32 bit mode.
Therefore, a chroot Sorcerer IA32 installation
will work if the SA so desires.
Also the IA32 libraries can be added to
/etc/ld.so.conf with appropriate consideration
in order to create an x86_64 box that is capable
of running IA32 executables such as pre-compiled
binary software
