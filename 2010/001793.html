xinetd was designed to be a replacement for inetd
the TCP UDP port superserver that is supposed
to listen on certain ports and then invoke a program
to handle inbound packets to a port.
xinetd works similar but is much fancier.
heimdal installs some programs that can be
run as services by xinetd provided that xinetd
is also installed.
Most SAs will not be using kerberos enabled services.
The error message can be ignored.

I am adjusting sorcery so that with the next release
that error message will not be seen.
Thanks for the report.
Like I mentioned above
in this situation there is no problem.
