I added a quick fix to redland.
If that grimoire is installed and,
redland, soprano, and kdelibs-v4 are recast
then that should be enough to activate
the nepomuk semantic desktop
after restarting kdm-v4 and logging in.

Also in the KDE control panel in advanced
under nepomuk/strigi server configuration advanced settings...
The amount of memory the nepomuk database should use can
be lowered to 50M.
I am thinking that 16M or 32M might be nice.
But if you use nepomuk much or just a little
then it can be adjusted here
or disabled completely which is normally my preference.

Jan reports sound is still broke on a KDE version 4.4.0 installation.
Yet audio is working for me.
So I do not have clues about what might be wrong.
Please report on the list if KDE audio is not working.
If we can gather more information
then perhaps we can arrive at a faster solution.

With the fix of KDE 4.4.0 to support the nepomuk stuff
it looks good to go.
Also KDE 4.4.0 no longer uses mysql through qt for nepomuk indexing.
It uses virtuoso-opensource.
And so that has changed.
Consequently, in the 2010-57 and later grimoire
some boxes that are not explicitly using mysql
or that the SA wants it installed in order to use
should be okay to have mysql dispelled.
However, it will cause qt-everywhere-opensource to recast
if mysql is removed.
