Making the change requires few steps,
if one knows precisely what those steps are.

Since newly deployed Sorcerer boxes boot from initramfs
I am considering an init-script that can fix the problem automatically.
But for now following the steps in this email is the easiest way to
fix the problem when booting using linux-modular and intiramfs.

Boxes using a carefully custom configured linux kernel will probably
not have the unexpected changes in block storage device names.
