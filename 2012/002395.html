For the possibility of being able to achieve mv command
where the source and destination exist on the same file system
which results in a rename operation;
/var/tmp/leech/
is now
/var/cache/download/

Likewise /var/tmp/failed/
is now
/usr/src/sorcery/failed/

The reason I selected /var/cache/download/
instead of a system that exists within /var/cache/sources/
is because /var/cache/sources/ might exist on a networked device,
such as NFS, CODA, ftpfs, sshfs file system.
Downloading to a networked file system is not desirable.

I want tasks such as source file moves and
the conclusions of failed casts to complete with minimal I/O.
Sorry for breaking long standing traditions and creating potential for
confusion.
