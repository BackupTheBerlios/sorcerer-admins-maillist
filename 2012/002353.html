My best guess is that if someone wants to try Sorcerer on a virtual machine
and encounters a problem that prevents booting
then they were probably thinking something like,
"Awesome, I am going to boot Sorcerer
on my 256M RAM virtual machine.
Here it goes."
And there it went straight into forever lag land.
I do not recommend booting the Sorcerer I/R
on a virtual machine with less than 1G of installed RAM
granted to the virtual machine.

The Sorcerer I/R has those significantly sized initramfs images
that are xz compressed.
Consequently, the amount of RAM required to load it is
Kernel, compressed images, space required to uncompress initramfs images,
space required to hold uncompressed files on rootfs,
space to run init-scripts and start services/daemons,
space to hold some additional files extracted from archives,
space to run installer, space to extract xz compressed archives, etc...
So basically, I am talking about plenty of space that is used
where almost none of it is re-taskable.
The RAM used to hold the compressed initramfs images
and decompress them can not be retasked by the kernel
until after the content of those xz compressed cpio images
is extracted and stored on rootfs which is a special type of tmpfs.
It is something like a money back offers where one must
spend $300 or more, send in receipt and completed form,
and then wait 6 months for a check to arrive by snail mail.
So better to go in with a full gigabyte of RAM
instead of trying to skirt across at the bare minimum
which risks a wipeout.

PAE is a requirement only for booting the IA32 I/R on an IA32 microprocessor
or an IA32 virtual machine emulated microprocessor.
If someone tries to boot the x86_64 I/R on an IA32 virtual machine
then a message will display that an x86_64 CPU is required to boot the kernel.
Whoever implement that, thanks.
The IA32 I/R can be booted on an x86_64 virtual machine, but is not
tested that way.

The minimum system specifications for using a Sorcerer I/R
is an 80686 microprocessor with PAE.
However, an x86_64 CPU, those with long mode,
are better choices for workstations and multimedia stations.
That is because displaying video can require
significant CPU and bus speed in order to process
even before being output to the memory of the graphics adapter.
