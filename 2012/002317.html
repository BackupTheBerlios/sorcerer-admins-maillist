I suppose it might seem that way.
After the first reboot to the fixed disk
provided that networking can be automatically configured
then the box will begin to automatically download, compile,
and unarchive software.
If for some reason this does not work properly
then try turning off the rapid feature
comment out
FEATURE[rapid]=1 in /etc/sorcery/local.config
That way only source tarballs will download.
Installation will require longer.
But that will circumvent a problem that could
exist in the archive cache if such a problem exists.

The "Sorcerer Sentient" installation is 10% from the I/R
and 90% from the network.
That is because the massive amount of data required in order
to provide firefox, kde, xorg, video players, fonts, and other good stuff
requires more space than a CDROM can provide.
Consequently, about an hour is required for the box
to complete a full installation after the first reboot.
Patience may and a networking connection are the main requirements.
Progress can be checked via # augur scry.
And the first virtual console will probably provide the option of
executing that automatically without having to login and invoke
the program using a shell prompt.

BTW, be certain that the I/R being used is the currently available I/R.
Older than current might not be usable.
I uploaded a new I/R last month.
